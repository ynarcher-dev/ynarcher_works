# [3-4-4] AC 1:1 멘토링 매칭 및 스케줄러 상세 요건서

본 문서는 전문가 멘토와 보육 스타트업 간의 1:1 자문 매칭 일정을 관리하는 '멘토링 스케줄러 탭'의 조작 화면, 공식 멘토링 마일스톤 일정 연동 및 유효성 검증 규칙, 시간 중복 감지 로직, 그리고 이원화된 예약 모드(`ADMIN_ONLY` / `STARTUP_FCFS`)에 따른 세부 트랜잭션 흐름을 정의합니다.

---

## 1. 멘토링 스케줄러 및 가용 슬롯 설정 UI

멘토링 탭은 날짜/시간별로 일정을 예약하고 확정 현황을 시각적으로 파악할 수 있는 스케줄러 그리드 뷰를 메인으로 삼습니다.

```
┌────────────────────────────────────────────────────────────────────────┐
│ [선택 날짜: 2026-07-06 ] [보기 모드: 일간 / 주간 / 월간 ]  [가용 슬롯 생성 +]│
│ ※ 해당 일자는 [1:1 멘토링 공식 일정: 2026-07-01 ~ 2026-07-31] 내에 있음  │
├────────────────────────────────────────────────────────────────────────┤
│ ■ 1:1 멘토링 스케줄러 그리드 (Timeline Grid)                            │
│ ┌─────────┬──────────────┬──────────────────┬────────────────────────┐ │
│ │ 시간대  │ 박멘토(특허) │ 이멘토(BM/마케팅) │ 최멘토(IR/투자)        │ │
│ ├─────────┼──────────────┼──────────────────┼────────────────────────┤ │
│ │ 10:00   │ [ A-바이오  ]│ [ (가용 슬롯)   ]│ [    (비어 있음)       ]│ │
│ │ ~11:00  │   (예약 완료)│   (선착순 신청대기)│                        │ │
│ ├─────────┼──────────────┼──────────────────┼────────────────────────┤ │
│ │ 11:00   │ [ (비어 있음)│ [ B-테크       ]│ [ C-케미컬            ]│ │
│ │ ~12:00  │              │   (예약 완료)    │   (예약 완료)          │ │
│ └─────────┴──────────────┴──────────────────┴────────────────────────┘ │
├────────────────────────────────────────────────────────────────────────┤
│ ■ [팝업] 1:1 멘토링 슬롯 배정 및 상세 정보 (Slot Config Modal)         │
│  - 일자 및 시간: 2026-07-06  [ 10:00 ] ~ [ 11:00 ]                     │
│  - 전문가 지정: [ 박멘토 x ]    - 대상 스타트업: [ A-바이오 v ]         │
│  - 진행 장소/방법: [ 본관 2층 소회의실 / 대면 v ]                          │
│  - 슬롯 상태: (o) 예약 완료(booked)  ( ) 가용 슬롯(available)           │
│  [배정 및 변경 사항 저장] [일정 취소]                                  │
└────────────────────────────────────────────────────────────────────────┘
```

## 2. 세부 기능 및 비즈니스 요건

### 2.1 드래그 앤 드롭 그리드 스케줄러 (Drag & Drop Scheduler Grid)
* **그리드 레이아웃**:
  - **가로축(X축)**: 본 보육 사업에 매핑된 전문가 멘토 목록 (`NETWORKS` 마스터 기반 참조).
  - **세로축(Y축)**: 1시간 단위의 시간대 타임라인 (09:00 ~ 18:00 기본 노출, 설정에 따라 30분 단위 분할 가능).
* **드래그앤드롭 슬롯 생성**:
  - 담당자가 빈 시간대 그리드 슬롯 영역을 마우스로 드래그(Drag)하여 일정 시간 블록(`matching_slots`)을 실시간으로 생성할 수 있습니다.
  - 생성된 블록을 마우스 더블클릭하면 [Slot Config Modal]이 팝업으로 나타나 세부 내역을 수정할 수 있습니다.

### 2.2 공식 멘토링 마일스톤 일정 연계 및 시간 범위 유효성 검증
멘토링 일정이 공식 프로그램 사업 기간을 벗어나서 임의로 예약되는 것을 원천 차단하는 유효성(Validation) 규칙입니다.

* **마일스톤 동기화 연동**:
  - 본 보육 사업의 마일스톤 달력([3_4_5_ac_milestones.md](./3_4_5_ac_milestones.md))에 카테고리가 `mentoring`(1:1멘토링)으로 지정된 공식 일정의 `시작일` 및 `종료일` 범위를 예약 가드 범위로 실시간 상속받습니다.
* **시간대 입력 통제 (UX Guard)**:
  - 스케줄러 상단 날짜 선택창(DatePicker) 클릭 시, **공식 마일스톤 범위를 벗어난 날짜들은 캘린더에서 비활성화(Disabled/회색 표기) 처리**하여 사용자가 클릭할 수 없도록 사전에 방지합니다.
* **백엔드 저장 차단 검증 규칙 (Backend validation)**:
  - 관리자가 수동으로 슬롯을 저장하거나 스타트업이 예약을 확정하는 시점에, 데이터의 `matching_date`가 공식 마일스톤 일정 `[start_date, end_date]` 범위에 포함되는지 한 번 더 검증합니다.
  - 범위를 벗어난 저장 요청의 경우, 트랜잭션을 롤백하고 **"선택한 날짜(2026-06-30)는 해당 사업의 1:1멘토링 공식 일정(2026-07-01 ~ 2026-07-31) 범위에 포함되지 않아 저장할 수 없습니다."**라는 팝업 경고와 함께 전면 차단합니다.

### 2.3 충돌 경고 알고리즘 및 오등록 방지 (Collision Detection)
* **충돌 감지 트리거**:
  - 관리자가 수동 배정 시 또는 스타트업이 예약을 시도하는 시점에, 백엔드 데이터베이스 상에서 시간 영역 중복 여부를 쿼리(Query)합니다.
* **검증 알고리즘**:
  - 새로 등록하고자 하는 슬롯의 `[시작 시각, 종료 시각]` 범위가 다음 조건 중 하나라도 만족하는 기존 슬롯이 있는지 검사합니다:
    1. 동일한 **전문가 UUID**를 가지며, 시간 범위가 중복되는 경우.
    2. 동일한 **스타트업 UUID**를 가지며, 시간 범위가 중복되는 경우.
* **예외 처리 UX**:
  - 중복이 발견될 경우, 저장이 전면 차단되며 화면 상에 붉은색 오류 경고창과 함께 중복 정보를 구체적으로 안내합니다.

### 2.4 매칭 모드별 세부 예약 트랜잭션 흐름
사업 최초 개설 시 [3_4_2_ac_overview.md](./3_4_2_ac_overview.md)에서 설정한 `mentoring_match_mode`에 따라 예약의 작동 방식과 권한이 완전히 이원화됩니다.

#### A. 관리자 직접 배정 모드 (`ADMIN_ONLY`)
* **조작 주체**: AC 사업 담당 임직원.
* **프로세스**:
  1. 관리자가 스케줄러 그리드에서 특정 전문가의 타임라인을 드래그하여 슬롯 생성.
  2. 팝업 창에서 대상 스타트업을 드롭다운에서 선택.
  3. [배정 및 변경 사항 저장]을 누르면 슬롯 상태가 즉시 `booked`(예약 완료)로 등록됨.
  4. 확정 시 해당 전문가와 스타트업 담당자 연락처로 알림톡(SMS)이 발송됩니다. 스타트업/전문가는 Guest 대시보드([3_9_workspace_guest.md](./3_9_workspace_guest.md))에서 확인만 가능합니다.

#### B. 스타트업 선착순 예약 모드 (`STARTUP_FCFS`)
* **조작 주체**: 전문가/관리자(슬롯 생성) 및 스타트업 Guest(예약 신청).
* **프로세스**:
  1. **가용 슬롯 생성**: 관리자가 전문가의 희망 가용 시간대를 취합하여 스케줄러 그리드에 슬롯을 대량 생성하되, 스타트업 매핑은 비워두고 상태를 `available`(선착순 신청 대기)로 설정하여 저장합니다. (생성 날짜는 반드시 위의 1:1멘토링 공식 일정 범위 내여야 함.)
  2. **슬롯 퍼블리싱**: 저장 즉시 해당 슬롯들은 스타트업 Guest 대시보드의 예약 콘솔([3_9_workspace_guest.md](./3_9_workspace_guest.md))에 실시간 노출되기 시작합니다.
  3. **선착순 예약 신청**: 스타트업 Guest가 비어 있는 가용 시간대 슬롯을 터치하고 [예약 확정]을 클릭하면, 해당 슬롯에 스타트업 UUID가 바인딩되며 상태가 즉시 `booked`(예약 완료)로 전환됩니다.
  4. **낙점 방식 트랜잭션(Concurrency Control)**:
     - 동시에 두 명의 스타트업이 동일한 `available` 슬롯을 선점하기 위해 [예약 확정]을 눌렀을 경우, 데이터베이스의 낙관적 락(Optimistic Lock)을 적용하여 **먼저 트랜잭션을 체결한 1인만 예약을 승인**하고, 차순위 신청자에게는 "이미 마감된 예약 슬롯입니다." 팝업을 띄우며 예약을 차단합니다.

### 2.5 멘토링 슬롯 라이프사이클 관리
* **completed (멘토링 완료)**:
  - 멘토링 예정 시각이 경과한 뒤, 멘토가 모바일에서 [상담일지 작성]을 완료하거나 관리자가 수동으로 '완료' 처리하면 상태가 변경됩니다.
  - 상태가 완료로 변경되는 즉시, 양 게스트의 Guest 화면 내 [피드백/평가 작성 폼]이 활성화됩니다.
