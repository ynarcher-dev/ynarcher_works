# [3-2] ADMIN (시스템 관리자) 워크스페이스 상세 기능 요건서

본 문서는 시스템 관리자가 사내 9개 내부 워크스페이스별 접근 권한(읽기/쓰기)을 유연하게 제어하고, 시스템 전반의 보안 트랙을 실시간 관제할 수 있도록 화면 요건과 비즈니스 흐름을 정의합니다.

---

## 1. 주요 화면 및 기능 명세

### 1.1 동적 메뉴 및 데이터 접근 권한 제어 콘솔
* **메뉴 권한 통제 매트릭스**
  * **기능**: 코딩 마이그레이션 없이, UI 상에서 각 등급별로 각 메뉴의 기본 CRUD 권한을 실시간 제어합니다.
  * **콘솔 구성**:
    * **역할 선택**: 관리자, 경영진, 매니지먼트(경영지원), 투자실, 사업부, M&A팀, 프로젝트팀, 스타트업, 전문가, 게스트 중 대상 역할을 선택합니다.
    * **메뉴별 토글 스위치**: 각 내부 워크스페이스(OFFICE, STARTUP, NETWORKS, AC, M&A/PE, PROJECT, FUND, MANAGEMENT, ADMIN)와 외부 서브도메인 앱 권한 도메인(GUEST) 옆에 **[읽기 권한 (Read)]** 및 **[쓰기 권한 (Write)]** 토글 스위치가 나열됩니다.
    * **동작**: 토글 스위치를 활성화/비활성화하고 [권한 적용] 버튼을 클릭하면, 해당 등급의 세션 권한 범위 및 프론트엔드 GNB 메뉴 노출 상태가 실시간 동적 갱신됩니다.

> [!IMPORTANT]
> 프론트 라우팅 기준의 권한 키는 `office`, `startup`, `networks`, `ac`, `fund`, `mna`, `project`, `management`, `admin`, `guest`입니다. 현재 일부 초기 DB enum·permission seed 및 권한 콘솔 설정에는 과거 `hub` 기준이 남아 있으므로, ADMIN 권한 콘솔은 위 10개 권한 키를 정본으로 노출하도록 정합화되어야 합니다.

### 1.2 시스템 보안 감사 로그 모니터 (Audit Log Monitor)
* **전사 중요 조작 및 접근 감사 로그 테이블**
  * **기능**: 전사 워크스페이스에서 발생한 비즈니스 중요 트랜잭션의 감사 추적 데이터를 제공합니다.
  * **수집 및 노출 항목**: 조작 사용자명, 대상 워크스페이스, 수행 액션(조회/수정/삭제/다운로드), 일시, 접속 IP 정보, 상세 변경 전/후 상태(JSON 데이터 대조 뷰어 팝업 제공).
* **첨부파일 다운로드 사유 로그 뷰 (Download Log View)**
  * **기능**: 사용자가 상세 뷰 첨부파일 목록에서 파일을 다운로드할 때 입력한 목적/사유 로그를 중점 관제합니다.
  * **주요 항목**: 다운로드 기안자, 소속, 대상 파일명, 입력된 상세 사유 텍스트, 다운로드 시각.

---

## 2. 관제 안전 장치 요건

* **재귀 잠금 방지 (Self-Lockout Prevention)**: 시스템 관리자가 본인의 `ADMIN` 워크스페이스 접근 권한이나 쓰기 권한을 비활성화하는 조작을 실행하려고 할 경우, "시스템 관리자 권한은 해제할 수 없습니다"라는 시스템 경고와 함께 적용이 원천 차단되도록 안전장치를 기획합니다.
