# [2] 핵심 비즈니스 시나리오 및 데이터 유기성 흐름 정의서

본 문서는 통합 Works 플랫폼 내의 각 워크스페이스(AC, M&A, FUND 등)가 고정된 직렬 단계나 순서에 얽매이지 않고, 중앙 마스터 데이터를 유기적으로 **상호 참조(Cross-Referencing)**하는 구조와 연동 규칙을 정의합니다.

---

## 1. 다각적 상호 참조 아키텍처 (Cross-Reference Architecture)

와이앤아처의 비즈니스 데이터는 순차적 단계(Step)가 아닌, 모든 정보가 서로 얽혀 있는 그물망식 **상호 참조 네트워크**로 존재합니다. 스타트업이 투자를 먼저 받고 보육 프로그램에 참가할 수도 있으며, 보육 없이 바로 M&A 딜의 대상으로 진입할 수도 있습니다.

```
       ┌─────────────────────────────────────────────────────────┐
       │          NETWORKS 마스터 (스타트업/전문가/협력사 데이터)    │
       └──────────────┬───────────────────┬──────────────────────┘
                      │                   │
               상호 참조                  상호 참조
                      │                   │
    ┌─────────────────▼─────────┐   ┌─────▼──────────────────────┐
    │     AC (보육 사업)         │   │       FUND (투자 펀드)      │
    │  - Program First 모듈      │   │  - 펀드별 포트폴리오 매핑   │
    │  - 모집/평가/멘토링/매칭    │   │                             │
    └─────────────────┬─────────┘   └─────┬──────────────────────┘
                      │                   │
               상호 참조                  상호 참조
                      │                   │
    ┌─────────────────▼───────────────────▼──────────────────────┐
    │              OFFICE (공통 업무 허브 및 캘린더)             │
    │  - 전문가 만족도 랭킹, 전사 크로스 검색, 통합 일정 캘린더   │
    └────────────────────────────────────────────────────────────┘
```

### 1.1 개별 비즈니스 데이터의 유기적 상호 참조 규칙

어느 시점에 데이터가 입력되든, 고유 식별자 매핑을 통해 전사 모든 영역에서 아래와 같은 유기적 연동 및 시각 효과가 즉시 발현됩니다.

#### A. NETWORKS 스타트업 마스터 ↔ AC 보육 프로그램 간 참조
* **연동 흐름**: `AC` 사업 담당자가 보육 프로그램의 모집(Recruitment)을 개설하면, 외부 스타트업이 GUEST 포털에서 지원서를 제출합니다. 이 신청서의 기본 정보(대표자명, 사업자번호 등)를 기준으로 `NETWORKS 스타트업 마스터`를 조회 및 연동합니다.
* **상호 반영**: 
  * 매핑 또는 임시 마스터 생성이 완료되면, 프로그램 상세 뷰의 '참가자 풀(Participant Pool)'과 NETWORKS 스타트업 마스터의 '보육 이력 패널'에서 프로그램 정보가 실시간 연동됩니다.
  * 참가 스타트업의 보육/신청 상태(`submitted` 지원 ➔ `screening` 심사중 ➔ `selected` 선정 ➔ `completed` 수료 / `dropped` 중도탈락)가 상호 참조되어, 어느 화면에서 상태를 변경하든 양방향 즉시 동기화됩니다.
* **개발 보안/운영 메타 데이터**:
  * **관련 사용자 권한**: 내부 임직원 (AC RW, NETWORKS RW), 외부 스타트업 (GUEST R/W - 본인 신청 및 본인 자료실 한정)
  * **민감 데이터 여부**: 보통 (이름, 이메일, 전화번호 등 개인정보 포함 ➔ 마스킹 처리 대상)
  * **필요 감사 로그**: 스타트업 마스터 정보 병합 및 소유권 변경 시 감사 로그 필수 기록
  * **외부 사용자 접근 여부**: 제한적 허용 (외부 스타트업은 본인 기업의 신청 정보 및 타임라인만 접근 허용, 타사 데이터 차단)
  * **다운로드/export 가능 여부**: 일반 조회 가능, 대량 엑셀 export 시 반출 권한 확인 및 사유 입력 필수

#### B. NETWORKS 스타트업 마스터 ↔ FUND 투자 조합 간 참조 (자사 투자 연동 강조)
* **연동 흐름**: `FUND` 워크스페이스에서 펀드를 통한 투자 집행 내역을 등록할 때, 대상 피투자사로 `NETWORKS`에 등록된 스타트업 정보를 연결합니다.
* **상호 반영**:
  * 투자실에서 집행한 투자 일자, 투자 금액, 지분율 데이터는 해당 스타트업 상세 뷰 내 '성장 지표' 섹션의 **투자 유치 내역**으로 자동 연동되어 표출됩니다.
  * 이때, 일반 사외 투자 유치 내역과 달리 **자사(와이앤아처)가 집행한 투자 건은 UI 상에서 차별화된 강조 효과**(특수 테두리 액센트 및 '자사 투자 연동' 배지 노출)를 부여하여 사용자가 직관적으로 구분할 수 있도록 렌더링합니다.
* **개발 보안/운영 메타 데이터**:
  * **관련 사용자 권한**: 내부 임직원 (FUND RW, 경영진 R, 최고 관리자 RW)
  * **민감 데이터 여부**: 높음 (조합 출자 정보, 자사 지분율, 투자 계약 조건 ➔ Restricted 등급)
  * **필요 감사 로그**: 투자 집행 내역 수정, 삭제, 지분율 변동 등록 시 감사 로그 필수 기록
  * **외부 사용자 접근 여부**: 허용 안 함 (외부 스타트업/전문가는 자사 투자 내역 및 조합 정보 조회 차단)
  * **다운로드/export 가능 여부**: 원칙적 불가 (투자실 담당 매니저 및 최고 관리자 승인 하에만 export 허용)

#### C. 전문가 멘토링/비즈니스 매칭 ↔ 상호 평가 및 OFFICE 랭킹 참조
* **연동 흐름**: `AC` 보육 프로그램 내에서 `NETWORKS` 마스터에 등록된 전문가와 스타트업을 매칭하여 `N:N 멘토링 관계` 또는 `1:1 비즈니스 매칭 슬롯`을 개설합니다.
* **상호 반영**:
  * 세션(멘토링/상담)이 완료되면 스타트업과 전문가는 모바일 Guest 화면을 통해 상호 만족도 평가 및 상담일지를 작성합니다.
  * **공통 평가 엔진 활용**: 평가 및 만족도 수집은 공통 평가 엔진(`evaluation_forms`)을 활용하며, 멘토/전문가는 스타트업을 대상으로 5대 지표(기술성, 사업 BM, 신뢰도 등)를 평가하고 스타트업은 멘토의 만족도 정량 점수(1~5점)를 제출합니다.
  * **OFFICE 랭킹 정렬**: 전문가들이 프로그램 전반에서 얻은 누적 만족도 평가 점수는 실시간으로 취합 및 평균 계산되어, `OFFICE` 대시보드 또는 관련 랭킹 보드에 **내림차순 정렬**로 자동 노출됩니다.
* **개발 보안/운영 메타 데이터**:
  * **관련 사용자 권한**: 내부 임직원 (AC RW, OFFICE R), 외부 전문가 (GUEST RW), 외부 스타트업 (GUEST RW)
  * **민감 데이터 여부**: 보통 (전문가 정성 평가지 내용 및 내부 만족도 피드백은 Restricted 등급으로 외부 비공개)
  * **필요 감사 로그**: 전문가 평가 항목 일괄 변경 및 배정 예외 처리 시 감사 로그 기록
  * **외부 사용자 접근 여부**: 허용 (본인에게 매핑된 평가 및 상담 폼에 한해 API 읽기/쓰기 허용)
  * **다운로드/export 가능 여부**: 평가 데이터의 다운로드는 권한이 부여된 임직원으로 제한

## 2. 중복 입력 방지 및 전사 공유 일정 동기화 기획

* **Program 통합 타임라인 및 전사 캘린더 동기화**:
  * 프로그램 내부에서 생성된 모든 일정(모집 일정, 평가 슬롯, OT 세션, 멘토링 회차, 비즈니스 매칭 슬롯 등)은 `Program 통합 타임라인`에 수집되며, 중요 마일스톤 일정은 전사 공통 일정 테이블로 실시간 동기화됩니다.
  * 타 부서나 경영지원실에서도 중복 입력 없이 `OFFICE` 캘린더를 통해 전사 일정을 한눈에 크로스 체크할 수 있습니다.
* **시간대 및 리소스 충돌 방지**:
  * 멘토링이나 비즈니스 매칭 슬롯 생성 시, 스타트업 및 전문가의 개인 타임라인, 동일한 장소/부스의 이용 시간이 겹치지 않도록 **시스템 충돌 검사** 및 **이동 버퍼 시간**(온라인 5분, 오프라인 10분 등)을 반영하여 일정을 조율합니다.
* **마스터 데이터 단일 변경 원칙 (Single Source of Truth)**:
  * 스타트업 물리 원장은 `public.startups`를, 전문가·투자사·기관·기업·대학·외주/거래처 등 외부 네트워크 정보는 `NETWORKS` 마스터를, 임직원 정보는 `MANAGEMENT` 인사 마스터를 참조합니다. 한 곳에서 정보가 변경되면 이를 참조하는 전사의 하위 워크스페이스 화면에 실시간 자동 반영되어 데이터 파편화를 방지합니다. `OFFICE`는 마스터를 소유하지 않으며, 이들 마스터 데이터를 통합 조회하는 업무 허브 역할만 수행합니다.

## 3. 정보 보완 및 사용자 경험(UX) 보안 가이드

* **상세 페이지 정보 섹션 표시/숨김 설정 (sections 메타데이터)**:
  * 스타트업 및 사업의 상세 화면은 다양한 정보 영역(주주 정보, 성장 지표, 마일스톤, 첨부파일 목록 등)으로 구성됩니다.
  * 사용자의 성격이나 관리 목적에 따라 불필요한 정보 영역은 화면 내에서 감출 수 있도록 **세션 노출 토글 스위치**를 제공합니다. 이 표시/숨김 상태 커스텀 값은 해당 데이터의 메타 정보에 저장되어 개별 사용자에게 지속적으로 맞춤형 뷰를 보장합니다.
* **첨부파일 목적 기반 다운로드 로그 감사 정책**:
  * 기업 보고서, 실사서, NDA 계약서 등 민감 문서를 포함한 첨부파일 목록에서 사용자가 다운로드 클릭 시 **[다운로드 목적/사유]** 입력을 필수적으로 요구하는 팝업이 활성화됩니다.
  * 사용자가 사유를 입력해야 다운로드가 승인되며, 다운로드한 사용자명, 일시, 대상 파일명, 입력된 사유가 전사 보안 로그(`Audit Log`)에 영구 기록되어 보안 사고를 사전에 방지합니다.
  * **개발 보안/운영 메타 데이터**:
    * **관련 사용자 권한**: 내부 임직원 (도메인별 read/write 권한자), 외부 게스트 (권한 허용 범위에 한함)
    * **민감 데이터 여부**: 높음 (실사 서류 및 NDA 계약서 원본)
    * **필요 감사 로그**: **필수** (다운로드 목적, 일시, IP, 대상자 정보)
    * **외부 사용자 접근 여부**: 한시적 허용 (임시 발급된 presigned URL 및 외부 스타트업 본인 업로드 건 한정)
    * **다운로드/export 가능 여부**: 허용하되 반드시 다운로드 사유 검증 팝업 및 API 미들웨어 세션 검증 필수
* **뉴스룸 실시간 뉴스 연동**:
  * 스타트업 상세 뷰 최하단에는 포털 API를 연동한 실시간 뉴스 피드 영역이 배치됩니다.
  * `STARTUP`/`NETWORKS`에 등록된 스타트업명을 키워드로 작동하여 수동 데이터 입력 없이 실시간 관련 뉴스 이슈가 노출되며, 외부 자동 동기화 영역이므로 수동 편집 버튼이나 최종 수정일 표시 요소를 노출하지 않아 UI를 심플하게 유지합니다.
