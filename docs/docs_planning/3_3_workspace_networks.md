# [3-3] NETWORKS (네트워크 관리) 워크스페이스 상세 기능 요건서

본 문서는 전사 공통으로 사용되는 핵심 인적 자원 마스터인 스타트업, 외부 전문가, 협력 파트너사 데이터를 체계적으로 관리하고, 대량의 엑셀 데이터를 원활하게 축적할 수 있도록 화면 요건과 비즈니스 흐름을 정의합니다.

> [!NOTE]
> 스타트업 마스터의 구분(투자·보육·발굴·기타) 라이프사이클, 담당자 권한 모델, 관리현황 적용 범위는 세부 요건서 [3_3_1_startup_pool_classification.md](./3_3_1_startup_pool_classification.md)에서 규정합니다.

> [!IMPORTANT]
> **등록자·담당자·관리자 3역할은 NETWORKS 8종 전체 공통 원칙입니다**(정의: [3_3_1 §4.0](./3_3_1_startup_pool_classification.md)). 요약하면 — **등록자**는 레코드를 처음 업로드한 사람(`created_by`, 불변)이고, **담당자**는 이 레코드를 수정할 권한을 가진 관리 주체입니다. NETWORKS의 모든 엔티티(전문가·협력사 포함)와 스타트업의 비투자 구분(발굴·보육·기타)은 담당자가 **공동관리**(NETWORKS 쓰기 권한자 누구나 수정, 이력은 기여자 로그로 누적)이며, **투자기업만** 특정 지정 담당자(리드1+지원N)를 갖습니다. **관리자**(`super_admin`)는 전역 우회 권한으로 위 두 축과 별개입니다. 화면에서 등록자와 담당자는 별개 항목으로 표기하고, 등록자를 담당자 자리에 폴백 표시하지 않습니다.
>
> 위 "담당자"(우리 레코드의 관리 주체)는 스타트업의 담당 임직원/PM·협력사 연락처 같은 **콘텐츠 필드로서의 "담당자명"**(대상 기업 측 사람)과는 다른 층입니다. 혼동을 피하기 위해 콘텐츠 필드는 "담당 임직원", "연락처 담당자" 등으로 구체화해 표기합니다.

## 1. 주요 화면 및 기능 명세

### 1.1 통합 디렉토리 관리 대시보드
* **삼단 분할 탭 레이아웃 (스타트업 / 전문가 / 협력사)**
  * **기능**: 마스터 데이터 성격에 따라 탭을 분리하여 목록을 정렬 및 렌더링하고, 각 탭 내에서 개별 데이터의 추가/수정/삭제 조작을 시작할 수 있습니다.
  * **목록 구성 요건**:
    * **스타트업 탭**: 기업명, 대표자명, 투자 유치 단계, 분야 분류 배지, 담당 임직원/PM 프로필 참조. 담당자 마스터는 `MANAGEMENT`의 임직원 정보와 연결합니다. (각 행에 '상세/수정' 버튼 배치)
    * **전문가 탭**: 전문가명, 소속 기관, 직책, 전문 자문 도메인 배지 목록. (각 행에 '상세/수정' 버튼 배치)
    * **협력사 탭**: 협력사명, 파트너 분류(정부기관/대학/VC/대기업 등), 담당자 이름 및 연락처. (각 행에 '상세/수정' 버튼 배치)
* **다차원 필터링 및 스마트 검색 바**
  * **기능**: 다양한 조건식을 중첩 적용하여 상세 타깃을 정밀 색인합니다.
  * **검색 필터**:
    * 분야별(바이오, AI, IT, 친환경 등), 지역별 필터.
    * 전문가의 멘토링 만족도 평균 평점(1.0~5.0) 범위 지정 필터.

### 1.2 개별 데이터 관리 콘솔 (Individual Editor Console)
* **신규 등록 및 정보 수정 모달**
  * **기능**: 각 마스터 데이터 탭에서 "신규 추가" 버튼을 누르거나 목록의 "상세/수정" 버튼을 누르면 나타나는 오버레이 팝업창입니다.
  * **입력 항목 및 유효성 검사**:
    * **스타트업**: 기업명(필수), 대표자명(필수), 사업자등록번호(필수, 중복 검사), 연락처, 이메일, 홈페이지, 투자 단계 등.
    * **전문가**: 이름(필수), 이메일(필수, 중복 검사), 연락처(필수), 소속 기관, 전문 분야(태그 입력).
    * **협력사**: 협력사명(필수), 사업자등록번호(선택), 파트너 분류, 담당자명, 연락처.
  * **변경 이력 기록**: 정보를 생성하거나 수정하여 저장할 때, 어떤 사용자가 언제 무엇을 변경했는지에 대한 감사 로그 데이터를 생성하여 `ADMIN` 워크스페이스에 전송합니다.


### 1.3 대량 데이터 엑셀 임포터 (Excel Data Importer)
과거의 파편화된 데이터 파일(엑셀, CSV)을 플랫폼으로 손쉽게 일괄 이관하기 위한 일방향 임포트 기능입니다.

* **임포터 파일 업로드 드롭존 (Drag & Drop Zone)**
  * **기능**: 사용자가 엑셀 양식을 다운로드하여 작성한 후 화면 중앙의 드롭존 영역에 파일 업로드(또는 탐색기 파일 선택)를 수행합니다.
* **실시간 데이터 유효성 검사 및 정합성 피드백 팝업**
  * **기능**: 엑셀이 업로드되는 즉시 클라이언트 백그라운드 분석을 통해 정합성을 자동 검사합니다.
  * **주요 검증 요건**:
    * 필수값 누락 (예: 기업명, 이메일, 전화번호 누락 여부).
    * 데이터 포맷 오류 (예: 잘못된 이메일 형식, 전화번호 자릿수 오류).
    * 기존 마스터 DB와의 중복 검사 (예: 이미 등록된 이메일 또는 휴대폰 번호 존재 여부).
  * **피드백 UX**: 유효하지 않은 데이터가 발견되면 즉시 업로드를 차단하고, **"엑셀 14번 행의 전화번호 포맷 오류", "25번 행의 이메일 중복"** 형태의 오류 리스트를 붉은색으로 명시하여 파일 수정 후 재업로드하도록 안내하는 팝업을 제공합니다.

## 2. 전사 마스터 허브(Single Source of Truth) 역할 규칙

* **수평적 데이터 상속**: 본 `NETWORKS`에 등록된 모든 인력 및 기업 정보는 고유 UUID 식별자를 가집니다.
* **타 워크스페이스 매핑 흐름**: AC 프로그램 참가 기업 매핑, FUND 포트폴리오 집행 등록, M&A 딜 대상 지정 시 별도로 텍스트를 기입하거나 신규 개체를 생성하지 않고, 본 NETWORKS의 마스터 데이터를 검색 선택하여 식별자를 바인딩하는 형태로 참조를 통일하여 전사 데이터 불일치를 원천 방지합니다.
