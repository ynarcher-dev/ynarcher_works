# [3-6] M&A (M&A팀) 워크스페이스 상세 기능 요건서

본 문서는 M&A팀 담당자가 인수합병(M&A) 딜 파이프라인 관리, 비밀유지문서 및 실사 자료 검토, 기업 매수/매도 매칭 업무를 기밀이 보장된 환경에서 수행할 수 있도록 화면 요건과 비즈니스 흐름을 정의합니다.

---

## 1. 주요 화면 및 기능 명세

### 1.1 M&A 딜 소싱 칸반 보드 (M&A Kanban Board)
* **칸반 구성**: 진행 중인 M&A 프로젝트(딜)를 진행 상태별로 구분된 세로 열(Column) 카드 보드로 출력합니다.
  * **진행 단계**: `소싱(Sourcing) ➔ 예비 실사(Pre-DD) ➔ 정밀 실사(DD) ➔ 조건 협상 ➔ 계약(Closing) ➔ 완료/무산`
* **카드 요약 정보**: 딜 대상 기업명, 딜 명칭, 담당 M&A 심사역, 거래 추정 가액, 최종 업데이트 경과일자, 보류 경고 배지.
* **조작 흐름**: 카드를 마우스로 드래그 앤 드롭하여 단계를 즉시 전환할 수 있습니다. 
  * 단계가 변경될 때마다 **[프로젝트 타임라인 로그]**에 변경 일시, 변경자, 이전 단계 ➔ 변경 후 단계가 자동 기록되어 이력 분석이 가능하도록 기획합니다.

### 1.2 딜 상세 프로필 및 보안 서류 검토
칸반 카드를 클릭하면 진입하는 핵심 실무 화면입니다.

* **NDA 및 비밀문서 체크리스트**
  * M&A의 중요 요건인 정보 보안을 위해 **비밀유지약정(NDA), 예비 양해각서(MOU), 주주 동의서** 등 필수 보안 문서의 업로드 상태를 체크리스트 형태로 제공합니다.
  * 담당 임직원은 문서를 검토 후 [검토 완료] 승인 버튼을 통해 문서의 상태를 잠금 처리할 수 있습니다.
* **정밀 실사(Due Diligence) 의견서 작성**
  * 실사 단계에서 외부 전문가나 M&A 담당자가 작성한 기업의 재무, 기술적 리스크 실사 보고서 파일을 관리합니다.
  * 해당 파일 다운로드 시 `[2] 핵심 비즈니스 시나리오`에 기획된 **[목적 기반 첨부파일 다운로드 로그]** 정책이 적용되어 사유 입력을 필수로 받습니다.

### 1.3 매수/매도 매칭 매트릭스 (Matching Matrix)
인수 희망 기업(Buyer)과 매각 희망 기업(Seller)의 매칭 정합성을 검토하는 도구입니다.

* **매칭 정합성 필터 콘솔**
  * 매수 측 희망 업종, 희망 인수 금액 상한선, 인수 목적 키워드와 매도 측 제시 조건을 대조합니다.
  * 정합성 매칭 알고리즘을 통해 매도 후보 기업들의 **적합도 백분율 점수**가 산출되어 상위순으로 정렬 노출됩니다.
* **매치 딜 생성**: 적합한 기업 쌍이 발견되면 매트릭스 화면에서 즉시 [신규 딜 생성] 버튼을 클릭해 소싱 단계의 칸반 카드로 즉시 인계할 수 있는 바로가기 흐름을 지원합니다.

---

## 2. 정보 격리 및 보안 특화 정책

* **엄격한 부서 격리**: M&A 딜 정보는 회사의 기밀에 속하므로, `ADMIN` 권한 제어 콘솔을 통해 M&A팀 담당자와 최고 관리자(Admin)를 제외한 타 부서(AC, 투자실, 프로젝트팀 등) 사용자의 읽기 권한을 기본 비활성화(`Hidden`) 처리합니다.
* **NETWORKS 연계**: 딜 대상 기업이나 인수 희망 기업의 기본 정보는 `NETWORKS` 디렉토리 마스터에서 검색하여 연결하며, 딜이 생성된 이후에도 NETWORKS에 등록된 정보와 동기화된 마스터 구조를 유지합니다.
