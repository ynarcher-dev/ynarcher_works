# [5] 공통 컴포넌트 규격 및 표준 디자인 시스템 정의서

본 문서는 와이앤아처 통합 Works 플랫폼의 시각적 일관성과 설계 효율성을 보장하기 위해, 컴포넌트 제작 시 적용해야 하는 **표준 글로벌 토큰(Radius, Shadow, Spacing)**과 **공통 UI 컴포넌트의 물리적 규격 명세**를 총망라하여 정의합니다.

---

## 1. 글로벌 디자인 토큰 (Global Design Tokens)

### 1.1 모서리 둥글기 (Border Radius)
과도하게 둥근 모서리를 지양하여 차분하고 전문적인 오피스/업무용 시스템 톤앤매너를 유지합니다.
* **`radius.sm` (4px)**: 텍스트 입력 필드(`Input`), 셀렉트 박스(`Select`), 배지(`Badge`), 체크박스/라디오.
* **`radius.md` (6px)**: 일반 버튼(`Button`), 페이지네이션 버튼, 드롭다운 팝오버 패널.
* **`radius.lg` (8px)**: 모달 다이얼로그(`Modal/Dialog`), 반복 카드 컴포넌트(`Card`), 전자결재 서류 정보 블록.
* **`radius.full` (9999px)**: 토글 스위치(`Switch`), 원형 사용자 프로필 아바타(`Avatar`).

### 1.2 그림자 효과 (Box Shadow)
기본적인 요소 간 구분을 위해 그림자를 남발하지 않으며, 테두리 선(Border, `#E5E5E5`)을 통한 분할을 원칙으로 하되 공중 플로팅이 필요한 컴포넌트에만 제한적으로 사용합니다.
* **`shadow.dropdown` / `shadow.popover`**: `0 4px 12px rgba(0, 0, 0, 0.08)` (드롭다운 메뉴, 툴팁, 자동완성 팝업)
* **`shadow.dialog`**: `0 8px 24px rgba(0, 0, 0, 0.12)` (전면 모달 팝업, 알림 다이얼로그)

### 1.3 여백 및 스페이싱 (Spacing Grid)
* 시스템 레이아웃은 **8px 그리드 시스템**을 표준으로 삼아 모든 컴포넌트의 마진(Margin)과 패딩(Padding)을 8의 배수로 적용합니다.
* **표준 여백 스케일**: `4px` (초소형) | `8px` (소형) | `12px` (중소형) | `16px` (기본 패딩) | `24px` (섹션 간 여백) | `32px` (대형 여백) | `48px` / `64px` (레이아웃 간격).

### 1.4 아이콘 스타일 규칙 (Icons)
* **표준 패밀리**: 선형(Stroke) 중심의 일관된 디자인 톤을 제공하는 `Lucide Icons` 또는 `React Icons (Hi / Hi2)` 계열을 통일하여 사용합니다.
* **기본 아이콘 크기 규격**:
  * **소형 (sm)**: `16px` (테이블 내 조작 버튼, 뱃지 내 아이콘, 헬퍼 텍스트)
  * **기본 (md)**: `20px` (사이드바 내비게이션, 일반 폼 레이아웃 보조 아이콘)
  * **대형 (lg)**: `24px` (모달 헤더 닫기 버튼, 메인 지표 카드 등 강조 영역)
* **스타일 통일**: 모든 아이콘은 `stroke-width: 2px`를 유지하여 시각적 무게감을 일치시킵니다.

## 2. 기본 조작 컴포넌트 명세 (Control Components)

### 2.1 버튼 (Button & IconButton)
버튼의 크기와 자형, 내부 여백은 다음과 같이 3단계 스펙으로 고정하여 통일합니다.

```
[ lg 버튼 (40px) ]  - 글씨: 16px (Medium)  |  패딩: 좌우 24px
[ md 버튼 (36px) ]  - 글씨: 16px (Medium)  |  패딩: 좌우 16px (기본값)
[ sm 버튼 (32px) ]  - 글씨: 14px (Regular) |  패딩: 좌우 12px
```

* **둥글기 규격**: `6px` (`radius.md`) 공통 적용.
* **아이콘 결합**: 버튼 텍스트의 좌/우에 아이콘이 결합되는 경우, 아이콘 크기는 `16px`로 고정하며 텍스트와의 간격은 `8px` (`gap-2`)을 유지합니다.
* **아이콘 전용 버튼 (IconButton)**: 텍스트 없이 아이콘만 노출되는 경우 스크린 리더 대응을 위해 반드시 `aria-label` 속성을 강제 선언해야 합니다.

### 2.2 입력 필드 (Input / Select / TextArea)
* **데스크톱 높이**: `36px` (`h-9`) 기본 고정.
* **둥글기 규격**: `4px` (`radius.sm`) 적용.
* **TextArea (멀티라인 입력)**: 최소 높이 `120px`를 보장하며, 우측 하단 리사이즈 제어는 상하 조절만 허용(`resize-y`)합니다.
* **웹 접근성 규칙**: 플레이스홀더를 입력 항목 명칭 대용으로 사용하지 않고, 입력 필드 좌측 또는 상단에 별도의 영구 텍스트 라벨을 무조건 노출합니다.

### 2.3 선택 컨트롤 (Checkbox / Radio)
* **박스 크기**: `16px * 16px` 고정.
* **둥글기**: 체크박스는 `4px` (`radius.sm`), 라디오 버튼은 완전 원형(`rounded-full`).
* **텍스트 연동**: 선택 컨트롤과 우측 라벨 텍스트 간의 거리는 `8px` (`gap-2`)를 보장하며, 터치/클릭 영역 확장을 위해 라벨 영역까지 클릭 인터랙션 범위를 매핑합니다.

### 2.4 토글 스위치 (Switch)
* **하우징 규격**: 가로 `36px` (w-9) | 세로 `20px` (h-5) | 둥글기 `rounded-full`.
* **스위치 버튼 (Thumb)**: `16px * 16px` 흰색 구체 | `checked=true` 시 우측으로 `16px` 슬라이드 번역 (`translate-x-4`).

### 2.5 아바타 (Avatar)
* **물리 크기 스케일**:
  * **초소형 (xs)**: `24px * 24px` (테이블 행 내 사용자 목록, 헬퍼 위젯 전용)
  * **소형 (sm)**: `32px * 32px` (상단바 프로필 트리거, 댓글 작성란 전용)
  * **기본 (md)**: `40px * 40px` (스타트업 대표 프로필 요약 카드, 계정 관리 기본 뷰)
  * **대형 (lg)**: `64px * 64px` ~ `80px * 80px` (상세 정보 메인 헤더 화면)
* **둥글기 규격**: 완전 원형(`rounded-full`) 공통 적용.
* **대체 텍스트 아바타 규칙**:
  * 이미지가 등록되지 않은 게스트/사용자의 경우, 이름의 첫 1글자(영문은 대문자 1글자)를 텍스트로 노출합니다.
  * 배경색은 사용자의 고유 ID 해시값을 기준으로 차분한 톤의 파스텔 배경색(`bg-gray-100`, `bg-blue-50`, `bg-brand-25` 등)을 매핑하여 인접한 아바타끼리 시각적으로 구분되도록 처리합니다. 텍스트 컬러는 배경색 대비 4.5:1 이상을 충족하는 어두운 차콜(`text-gray-800`)을 적용합니다.

## 3. 구조 및 데이터 컴포넌트 명세 (Structure & Data Components)

### 3.1 데이터 테이블 (Data Table)
대량의 행 데이터 판독 효율과 웹 접근성을 동시에 확보하기 위한 테이블 조판 가이드라인입니다.

* **테이블 래퍼**: 테이블 컴포넌트는 무조건 `border border-gray-200` 테두리로 둘러싸인 라운드 박스(`radius.lg`) 내에 렌더링되며, 모바일 대응을 위해 가로 스크롤 영역(`overflow-x-auto`)을 확보합니다.
* **머리글 영역 (THead)**:
  * **높이 규격**: 세로 `36px` (`h-9`) | **배경**: `bg-gray-50` (`#F7F7F7`).
  * **머리글 셀 (`th`)**: 폰트 `12px` (`text-xs`), 색상 `#737373` (`gray.500`), 좌우 패딩 `12px` (`px-3`).
  * **정렬**: 일반 글자형 속성은 **좌측 정렬(Left)**, 수치/금액 등 수량 관련 데이터는 **우측 정렬(Right)**을 부여합니다.
* **본문 영역 (TBody)**:
  * **높이 규격**: 데이터 행 세로 높이 `44px` (`h-11`) 고정.
  * **본문 셀 (`td`)**: 폰트 `14px` (`text-gray-800`), 좌우 패딩 `12px` (`px-3`).
  * **정렬 정합성**: 우측 정렬된 수치 컬럼에는 자릿수 선이 세로로 정확히 일치하여 정렬을 한눈에 판단할 수 있도록 **`tabular-nums` CSS 속성을 무조건 적용**합니다.
* **인터랙션**:
  * 클릭 가능한 행은 `cursor-pointer hover:bg-gray-50`을 부여합니다.
  * 선택 완료(Selected)된 행은 브랜드 연화 레드의 배경인 `bg-red-50` (`#FFF5F4` 계열)로 강조 하이라이트합니다.

### 3.2 리스트 페이지네이션 (Pagination)
* **중앙 정렬 번호 리스트**:
  * 테이블 하단에 `16px` (`mt-4`) 거리를 두고 배치하며 중앙에 번호 목록을 가로 정렬합니다.
  * 페이지 뱃지 크기: `32px * 32px` (`h-8 min-w-8`), 모서리 `6px` (`radius.md`).
  * 활성화 페이지는 테두리 없이 브랜드 레드 배경(`bg-brand`)에 `text-white`로 노출하고, 비활성 번호는 `text-gray-700`과 `hover:bg-gray-100`을 적용합니다.
* **퀵 점프 이동**:
  * 이전(◀) 아이콘 왼쪽에 **맨 앞으로 이동(⏮)** 버튼을 결합하고, 다음(▶) 아이콘 오른쪽에 **맨 뒤로 이동(⏭)** 버튼을 필수로 탑재합니다.
* **우측 직접 페이지 이동창 (Go-To-Page)**:
  * 페이지네이션 가장 우측(`absolute right-0`)에 별도의 "페이지 이동" 텍스트와 가로 `64px` 크기의 숫자 입력창(`InputNumber`)을 결합하여, 사용자가 원하는 페이지 번호를 직접 입력하고 엔터를 눌러 바로 이동할 수 있게 합니다.

### 3.3 탭 메뉴 (Tabs)
* **높이**: `40px` (`h-10`) 기본 고정.
* **스타일 규칙 (Underline Tab)**:
  * 탭은 카드 형태가 아닌 하단 라인형 방식을 기본으로 사용합니다.
  * 탭 하단 전체를 가로지르는 `1px` 두께의 회색 가이드 라인(`border-b border-gray-200`)을 배치합니다.
  * **활성 탭**: 폰트 두께 `500` (Medium), 글씨는 브랜드 레드(`text-brand`), 탭 하단에 `2px` 두께의 브랜드 레드 강조 테두리(`border-b-2 border-brand`)를 추가합니다.
  * **비활성 탭**: 폰트 두께 `400` (Regular), 글씨 `text-gray-700`, 호버 시 `hover:text-gray-900` 적용.

### 3.4 배지 및 태그 (Badge / Tag)
* **물리 크기**: 세로 `22px` 내외로 소형 설계, 둥글기 `4px` (`radius.sm`) 적용.
* **상태 매핑 4종**:
  * **완료/성공 (Success)**: 청색 계열 (글씨 `#1D4ED8` | 배경 `#EFF6FF` | 보더 `#BFDBFE`)
  * **대기/주의 (Warning)**: 황색 계열 (글씨 `#92400E` | 배경 `#FFFBEB` | 보더 `#FDE68A`)
  * **진행/정보 (Info)**: 초록 계열 (글씨 `#166534` | 배경 `#F0FDF4` | 보더 `#BBF7D0`)
  * **취소/중단 (Danger)**: 적색 계열 (글씨 `#9F170D` | 배경 `#FFF5F4` | 보더 `#FAD3CF`)

### 3.5 다이얼로그 모달 (Dialog / Modal)
* **너비 규칙**: 정보량에 따라 세분화합니다.
  * **소형 (sm)**: `400px` (단순 확인, 안내, 알림용)
  * **중형 (md)**: `600px` (일반 폼 데이터 추가/수정용)
  * **대형 (lg)**: `800px` ~ `1000px` (상세 정보 테이블 매핑 조회, 대규모 상세 폼)
* **모서리 둥글기**: `8px` (`radius.lg`) 적용.
* **그림자**: `shadow.dialog` (`0 8px 24px rgba(0, 0, 0, 0.12)`) 적용.
* **푸터 버튼 정렬 규칙**:
  * 모달 하단 푸터(Footer) 영역의 조작 버튼은 항상 **우측 정렬**을 원칙으로 합니다.
  * **보조 버튼(취소, 닫기 등)을 좌측**에 배치하고, **주요 핵심 액션 버튼(저장, 등록 등)을 가장 우측**에 나란히 배치(`gap-2`)하여 사용자의 결정을 돕습니다.

### 3.6 아코디언 (Accordion)
* **구조 및 여백**:
  * **헤더 (Header)**: 높이 `44px` (`h-11`) | 좌우 패딩 `16px` (`px-4`) | 폰트 `14px` Medium (`text-gray-800`).
  * **본문 패널 (Panel)**: 상하 패딩 `12px` (`py-3`), 좌우 패딩 `16px` (`px-4`) | 폰트 `14px` Regular (`text-gray-600`).
* **모션 및 아이콘 규칙**:
  * 아코디언 헤더 우측 끝에 펼침 상태를 지시하는 방향 표시 화살표 아이콘(Chevron, `16px`)을 필수로 결합합니다.
  * 열린 상태(`open=true`)일 경우 화살표는 위를 향하며(`rotate-180`), 닫힌 상태(`open=false`)일 경우 아래를 향합니다. 이 전환 과정은 `transform duration-200 ease-in-out` 트랜지션을 적용해 회전 애니메이션을 표현합니다.
  * 본문 영역의 슬라이드 다운/업 동작은 CSS 높이 트랜지션(`transition-[max-height] duration-200 ease-in-out`)을 적용해 부드러운 개폐 효과를 줍니다.

### 3.7 브레드크럼 (Breadcrumb)
* **컴포넌트 레이아웃**:
  * 헤더 및 메인 타이틀 영역 바로 위(Top)에 배치하여 사용자의 현재 탐색 위치를 안내합니다.
  * 각 경로 노드는 가로 정렬(`flex items-center gap-2`) 형태로 나열됩니다.
* **타이포그래피 및 가이드선**:
  * **경로 노드 텍스트**: `12px` (`text-xs`) 규격 적용.
  * **구분선 아이콘**: 경로 노드 사이에 쉐브론(`>`) 또는 슬래시(`/`) 형태의 연한 그레이 아이콘(`12px`, `text-gray-300`)을 배치합니다.
  * **이동 가이드**:
    * 이전 상위 경로 노드는 클릭 가능한 링크 형태이며, 호버 시 언더라인 및 활성 컬러(`hover:text-brand hover:underline`)를 적용합니다.
    * 현재 머무르고 있는 마지막 경로 노드는 링크 링크가 비활성화되며, 뮤트 그레이(`text-gray-400`)로 표기하여 현재 위치임을 인지시킵니다.

## 4. AC Program 운영 특화 컴포넌트 후보

AC 워크스페이스의 기능 모델은 `matching` 레퍼런스의 운영 감각을 가져오되, 화면 구성은 Works 공통 디자인 시스템을 따른다. 따라서 아래 컴포넌트는 별도 UI 복제가 아니라 기존 AppShell, PageHeader, Tabs, DataTable, Drawer, Modal, StatusBadge, SplitView 패턴을 조합하는 방식으로 정의한다.

### 4.1 ProgramCard / ModuleCard
* **ProgramCard**는 Program 목록과 대시보드에서 사용하며, Program명, 상태, 기간, 담당자, 활성 모듈 수, 미해결 운영 이슈 수를 표시한다.
* **ModuleCard**는 Program 상세의 모듈 보드에서 사용하며, `module_type`, `module_status`, `participation_mode`, 진행률, 마지막 업데이트 시각, 바로가기 액션을 포함한다.

### 4.2 EvaluationSplitView / FileSubmissionPanel
* **EvaluationSplitView**는 서면평가, 대면평가, 데모데이 평가에서 사용한다.
  * 좌측에는 신청서, PDF, 발표자료, 기업 요약 정보를 표시한다.
  * 우측에는 평가표, 점수 입력, 의견 입력, 임시저장/제출 액션을 배치한다.
* **FileSubmissionPanel**은 모집 신청, 발표자료 제출, 세션 자료 제출에서 사용하며, 파일 상태, 제출 마감, 검토 상태, 다운로드 권한을 함께 표시한다.

### 4.3 ScheduleSlotGrid / AssignmentMatrix
* **ScheduleSlotGrid**는 멘토링 회차, 비즈니스 매칭 슬롯, 대면평가 발표 시간표에서 사용한다.
* **AssignmentMatrix**는 심사위원-기업, 멘토-멘티, 전문가-상담 기업의 N:N 또는 1:1 배정 상태를 한 화면에서 확인하도록 구성한다.
* 두 컴포넌트 모두 시간 충돌, 배정 중복, 권한 범위 위반을 StatusBadge와 Inline Alert로 표시한다.

### 4.4 CounselingLogEditor / KpiSummaryPanel / ExportColumnPicker
* **CounselingLogEditor**는 멘토링 및 비즈니스 매칭 상담일지 작성에 사용하며, 요약, 이슈, 조언, 액션아이템, 후속 일정 필드를 기본으로 가진다.
* **KpiSummaryPanel**은 Program KPI와 모듈별 운영 지표를 카드형 요약으로 표시한다.
* **ExportColumnPicker**는 Excel 다운로드 전 컬럼 그룹, 마스킹 정책, 다운로드 사유를 선택하게 하며, Export 권한이 없는 사용자에게는 비활성 상태로 노출한다.

### 4.5 GuestTaskCard / MobileActionBar
* **GuestTaskCard**는 GUEST 포털의 오늘 할 일, 평가 제출, 예약 확인, 출석 체크, 상담일지 작성, 만족도 제출을 표시한다.
* **MobileActionBar**는 모바일 GUEST 화면 하단에 고정되는 주요 액션 영역이며, 한 화면에 하나의 핵심 제출/확정 액션만 강조한다.
