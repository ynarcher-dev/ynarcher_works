# [8] 전역 레이어 계층 Z-Index 규격서

본 문서는 와이앤아처 통합 Works 플랫폼 내의 수많은 겹침 요소(드롭다운, 사이드바, 상단바, 모달, 팝오버 등)가 화면에 배치될 때, 레이어 간의 시각적 간섭 및 가려짐 버그를 방지하고 일관된 깊이감(Depth)을 유지하기 위한 **전역 Z-Index 스케일 및 적응 규칙**을 정의합니다.

---

## 1. Z-Index 설계 원칙 (Layering Principles)

*   **상대적 계층 구조 (Relative Depth)**: 화면의 물리적인 의미에 따라 Z-Index의 상대적 높이를 지정합니다. 사용자의 포커스가 가장 늦게 이동하고 화면을 덮는 일시적 요소일수록 높은 계층을 배정받습니다.
*   **간격 확보 (Safe Margins)**: 차후 새로운 레이어 요소가 추가되더라도 전역 설정을 깨뜨리지 않고 끼워 넣을 수 있도록, Z-Index 값 사이에 일정한 임의 안전 간격을 둡니다.
*   **컴포넌트 독립성 (Component Isolation)**: 컴포넌트 내부에서만 쓰이는 국소적 Z-Index 값(예: 활성 버튼 테두리 등)은 전역 스케일에 침범하지 않도록 `z-10` 미만의 로컬 값으로 소화합니다.

---

## 2. 전역 Z-Index 스케일 테이블 (Global Z-Index Scale)

전체 플랫폼 웹 애플리케이션에서 공통으로 강제 매핑해야 하는 Z-Index 단계입니다.

| 토큰명 | Z-Index 값 | 주요 대상 및 용도 설명 |
| :--- | :---: | :--- |
| **`z.base`** | `0` | 기본 본문 텍스트, 카드 컴포넌트, 일반 문서 흐름 |
| **`z.sticky`** | `10` | 테이블 헤더 고정 (`sticky top-0`), 본문 내 고정 배너/패널 |
| **`z.dropdown`** | `100` | 입력 필드 자동완성, 드롭다운 메뉴, 툴팁, 커스텀 셀렉트 박스 팝오버 |
| **`z.navbar`** | `200` | **상단바 (Topbar)**. 본문 스크롤 시 본문 콘텐츠보다 항상 위에 노출됨 |
| **`z.sidebar`** | `300` | **좌측 사이드바 (Sidebar)** 및 모바일 뷰포트의 드로어(Drawer) 컨테이너 |
| **`z.overlay`** | `1000` | 모달/다이얼로그의 배경을 어둡게 차단하는 **딤 레이어 (Backdrop)** |
| **`z.modal`** | `1010` | 전면 화면 위에 뜨는 **실제 모달 다이얼로그 (Dialog / Modal Body)** |
| **`z.toast`** | `2000` | 시스템 전역 경고 및 조작 피드백 **토스트 (Toast / Snackbar)** |

---

## 3. 레이어 결합 및 간섭 방지 규칙

### 3.1 모달 내부에서의 드롭다운 제어
*   모달 다이얼로그(`z-modal: 1010`) 내부에 위치한 드롭다운이나 셀렉트 박스를 클릭할 경우, 해당 드롭다운 팝오버는 모달 바디보다 위에 떠야 합니다.
*   따라서, **포털(Portal)이나 팝오버(Popover) 라이브러리를 통해 화면 루트(Root)에 직접 렌더링되는 드롭다운 요소는 모달의 Z-Index 영역보다 높게 적용**(`z-index: 1100` 이상 또는 모달 컨텍스트 내 적응)되도록 동적으로 제어되어야 합니다.

### 3.2 상단바와 사이드바의 겹침 규칙
*   **데스크톱 뷰포트**: 좌측 사이드바(`240px` 고정)와 상단바(`56px` 높이)는 겹치지 않고 본문 영역을 공유합니다.
*   **모바일/태블릿 뷰포트**: 햄버거 메뉴를 눌러 좌측에서 미끄러져 들어오는 **사이드바 드로어(`z-sidebar: 300`)는 상단바(`z-navbar: 200`)를 완전히 덮어야 합니다.** 따라서 모바일 사이드바 드로어가 상단바보다 우선순위가 높은 레이어 위치를 점합니다.

---

## 4. Tailwind CSS 설정 (tailwind.config.js) 매핑 가이드

프로젝트 코드 작성 시 임의의 숫자(ex: `z-[999]`, `z-[9999]`)를 스타일링에 직접 대입하는 것을 금지하며, 아래와 같이 Tailwind Config에 별도의 토큰 명세로 등록하여 규격화된 클래스를 호출해 사용합니다.

```javascript
// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      zIndex: {
        'base': '0',
        'sticky': '10',
        'dropdown': '100',
        'navbar': '200',
        'sidebar': '300',
        'overlay': '1000',
        'modal': '1010',
        'toast': '2000',
      }
    }
  }
}
```

*   **실제 코드 호출 예시**:
    *   `<header class="sticky top-0 z-navbar">...</header>`
    *   `<div class="fixed inset-0 z-overlay bg-black/40"></div>`
