# [4] 브랜드 컬러 시스템 및 웹 접근성 규정 정의서

본 문서는 와이앤아처 통합 Works 플랫폼의 브랜드 정체성(CI)을 일관성 있게 전달하고, 정보의 가시성을 높이며, **한국형 웹 콘텐츠 접근성 지침(KWCAG 2.2)**의 명도 대비 규정을 완벽하게 만족하기 위한 **표준 컬러 시스템(Color System)**을 정의합니다.

---

## 1. 브랜드 아이덴티티 컬러 (Brand Colors)
와이앤아처의 CI Red(`#E22213`)는 **로고와 인쇄물 등 브랜드 표기 전용**으로 두고, 화면 UI의 액센트는 딥 네이비 팔레트가 담당합니다. 고채도 적색이 활성 탭·배지·CTA로 반복 노출되며 발생하던 시각 피로를 제거하고, 투자 심사와 수치 중심 화면에 요구되는 신뢰감을 확보하기 위한 결정입니다.

> [!NOTE]
> 브랜드가 적색 계열이 아니게 되면서, 기존에 브랜드 램프를 빌려 쓰던 **Danger(파괴성 액션·검증 실패) 용도는 별도의 `danger` 램프로 분리**되었습니다. 상세 규격은 3절과 5절을 참조합니다.

## 2. 기본 무채색 (Neutral Gray Scale) Palette
화면의 전체적인 균형과 가독성을 잡는 무채색 톤으로, 눈의 피로를 유발하는 완전한 검은색(`#000000`)의 사용을 지양하고 브랜드 네이비와 같은 온도의 차분한 쿨 슬레이트 톤을 적용합니다.

> [!NOTE]
> 텍스트 단계(400~900)와 경계선 단계(100~300)는 화면이 전반적으로 뿌옇게 보이던 문제를 해소하기 위해 재조정된 값입니다. 특히 `gray.400`은 규정상 비활성·플레이스홀더 전용이지만 실제 구현에서 메타 데이터 텍스트로 가장 많이 사용되고 있었고 명도 대비가 2.5대 1에 불과했습니다. 1차로는 토큰 값 자체를 어둡게 옮겨 400 단계까지 KWCAG AA를 충족시키고, 이웃 단계와의 명도 간격을 다시 벌렸습니다.

> [!IMPORTANT]
> **사용처 정합 완료(2026-07-21).** 위 1차 조치는 토큰 값만 옮긴 미봉책이었고, 한 토큰이 "본문 메타 텍스트"와 "비활성·플레이스홀더"라는 상반된 두 역할을 겸하는 구조적 문제가 남아 화면이 계속 흐려 보였습니다. `apps/works`·`apps/guest`·`packages/ui`의 회색 텍스트 416개소를 역할별로 전수 분류해 다음과 같이 재배치했습니다.
>
> * **메타 텍스트·라벨 240개소** → `gray.600`(본문 표준색)으로 승격. 날짜·건수·파일 크기·테이블 헤더·폼 필드 라벨 등 실제로 읽히는 정보가 대상입니다.
> * **빈 상태 안내 문구** → `gray.500`. 읽히되 강조되지 않는 단계입니다.

>
> 이후 회색 텍스트를 새로 쓸 때는 **"약하게 보이고 싶다"가 아니라 "이 텍스트의 역할이 무엇인가"를 기준으로** 단계를 고르십시오. 사용자가 읽어야 하는 정보라면 `gray.600` 이상이며, `gray.400`은 플레이스홀더·비활성 전용입니다.

## 3. 상태 신호색 (Semantic Colors) 규칙
시스템 내의 다양한 업무 상태를 시각적으로 직관적으로 전달하는 컬러 가이드입니다. 텍스트 판독성 강화를 위해 **텍스트(Default) - 배경(Subtle) - 보더(Border)**의 3단계 대비 구조를 유지하며, 토스(Toss) 디자인 스타일을 참조하여 산뜻하고 명확한 색상으로 제공합니다.

### 3.1 세만틱 컬러 적용 매트릭스

#### ① 성공 및 완료 (`Success` ➔ 초록 계열)
* **텍스트**: `#059669` | **배경**: `#ECFDF5` | **보더**: `#D1FAE5`
* **주요 적용**: 기안 결재 완료, 보육 스타트업 최종 선정, 펀드 결성 완료, 투자 집행 완료 등 완료 및 정상 성공 상태.

#### ② 주의 및 대기 (`Warning` ➔ 황색 계열)
* **텍스트**: `#D97706` | **배경**: `#FFFBEB` | **보더**: `#FEF3C7`
* **주요 적용**: 서류 반려, 기한 임박 안내, 승인 검토 대기, 미확인 요청 등 대기 및 주의 상태.

#### ③ 진행 및 정보 (`Info` ➔ 청색 계열)
* **텍스트**: `#0064FF` | **배경**: `#F0F5FF` | **보더**: `#E0EBFF`
* **주요 적용**: 프로그램 보육 진행 중, 멘토링 매칭 진행 중, 업무 프로젝트 진행 중, 임시 데이터 표기 등 일반적인 진행 및 정보 상태.

#### ④ 취소, 중단 및 위험 (`Danger` ➔ 적색 계열)
* **텍스트**: `#EF4444` | **배경**: `#FFF5F5` | **보더**: `#FFE2E2`
* **주요 적용**: 보육 사업 취소/중단, 투자 계약 반려/철회, 전자결재 문서 회수, 심사 탈락, 차단 상태.
* **버튼·보더 전용 심화 단계**: 흰 글씨를 얹는 파괴성 액션 버튼과 검증 실패 보더는 위 텍스트색(`#EF4444`)으로는 명도 대비가 부족하므로, 아래 심화 단계를 사용합니다. (브랜드 액센트가 네이비로 전환되기 전 적색 램프의 값을 그대로 승계한 것으로, Danger 표현의 기존 인상은 변하지 않습니다.)
  * **danger.700** `#B31A0F`: Danger 버튼 기본색, 입력 필드 오류 보더
  * **danger.800** `#9F170D`: Danger 버튼 Hover, 취소/중단 상태의 텍스트 색상
  * **danger.900** `#86130B`: Danger 버튼 Active

## 4. 웹 접근성(KWCAG 2.2) 컬러 준수 수칙

### 4.1 텍스트 명도 대비 보장
* 화면에 렌더링되는 모든 정보성 텍스트는 배경색 대비 **최소 명도 대비 4.5대 1 이상**을 유지해야 합니다. (단, 18pt/24px 이상의 대형 텍스트는 3대 1 이상).
* 보조 텍스트로 쓰이는 `#4A5361`(gray.600)은 명도 대비 약 **7.8대 1**로 기준을 충족하며, 포인트 강조용 `#1F3A5F`은 **11.5대 1**로 기준을 크게 상회합니다.
* 비활성 텍스트나 플레이스홀더용 `#6E7683`(gray.400)는 재조정을 거쳐 명도 대비 약 **4.6대 1**로 기준을 충족합니다. 다만 위계상 가장 약한 단계이므로 **본문 핵심 정보에는 단독으로 사용하지 않고 보조 설명문을 병행**하여 제공합니다.

### 4.2 색상 단독 구분 지양 원칙
* 정보 취약계층(적록색약 등)이 색상만으로 업무 상태를 혼동하지 않도록 제어합니다.
* **필수 규정**: 성공, 대기, 중단 등을 표현할 때 단순히 뱃지의 컬러(파랑, 노랑, 빨강)만 노출해서는 안 되며, **반드시 상태 텍스트 라벨("완료", "대기", "취소") 또는 형태가 다른 아이콘을 1:1로 함께 노출**하여 조판해야 합니다.

## 5. 컴포넌트별 인터랙션 컬러 규칙 (Component Interaction Rules)
플랫폼의 대표적인 인터랙션 컴포넌트인 버튼과 입력 필드의 상태별 컬러 규격을 정의하여 일관된 조작감을 제공합니다.

### 5.1 버튼 (Button) 컴포넌트
버튼은 역할(Variant)에 따라 아래의 테이을윈드 및 브랜드 테마 토큰 컬러 규격을 엄격히 준수합니다.

* **Primary (주요 액션 — 등록, 저장, 결재상신 등)**:
  * **기본 (Default)**: `bg-brand` (`#1F3A5F`) / 글씨 `text-white`
  * **호버 (Hover)**: `bg-brand-600` (`#1A3252`)
  * **클릭 (Active)**: `bg-brand-700` (`#152944`)
* **Secondary (보조 액션 — 취소, 이전, 닫기 등)**:
  * **기본 (Default)**: `bg-gray-100` (`#EDEFF2`) / 글씨 `text-gray-800` (`#2B313A`)
  * **호버 (Hover)**: `bg-gray-200` (`#DFE2E7`)
  * **클릭 (Active)**: `bg-gray-300` (`#CBD0D8`)
* **Outline (테두리 액션 — 파일 다운로드, 목록 필터링 등)**:
  * **기본 (Default)**: `border-gray-300` (`#CBD0D8`) 테두리 + `bg-white` / 글씨 `text-gray-800`
  * **호버 (Hover)**: `bg-gray-50` (`#F5F6F8`)
  * **클릭 (Active)**: `bg-gray-100` (`#EDEFF2`)
* **Ghost (텍스트 액션 — 테이블 행 제어, 간편 링크 등)**:
  * **기본 (Default)**: 배경 없음 / 글씨 `text-gray-700` (`#39404B`)
  * **호버 (Hover)**: `bg-gray-100` (`#EDEFF2`)
  * **클릭 (Active)**: `bg-gray-200` (`#DFE2E7`)
* **Danger (파괴성 액션 — 삭제, 철회, 기각 등)**:
  * **기본 (Default)**: `bg-danger-700` (`#B31A0F`) / 글씨 `text-white`
  * **호버 (Hover)**: `bg-danger-800` (`#9F170D`)
  * **클릭 (Active)**: `bg-danger-900` (`#86130B`)

### 5.2 입력 필드 (Input / Select / TextArea)
사용자가 데이터를 입력하는 폼 요소는 각 포커스 및 검증 실패 피드백이 실시간으로 제공되어야 합니다.

* **기본 상태 (Default)**:
  * `bg-white` 배경 + `border-gray-300` (`#CBD0D8`) 테두리 / 글씨 `text-gray-800` (`#2B313A`)
  * 플레이스홀더: `text-gray-400` (`#6E7683`)
* **포커스 상태 (Focus)**:
  * 텍스트 입력 활성화 상태. 테두리는 브랜드 포인트 컬러인 `border-brand` (`#1F3A5F`)로 강조하며, 외곽에 투명도 30%의 아웃라인 링(`focus-visible:ring-brand/30`)을 둘러 포커스 위치를 선명히 표시합니다. (기본 outline은 제거)
* **비활성 상태 (Disabled)**:
  * 입력 및 수정 불가 상태. `bg-gray-50` (`#F5F6F8`) 연한 그레이 배경에 `border-gray-200` (`#DFE2E7`) 테두리 / 글씨 `text-gray-400` (`#6E7683`)
  * 마우스 포인터는 금지 마크(`cursor-not-allowed`)를 노출하여 사용자의 오인 조작을 방지합니다.
* **오류 상태 (Invalid)**:
  * 필수값 누락 등 유효성 검사 실패 상태. 테두리 및 포커스 링이 경고용 붉은색 (`border-danger-700` / `#B31A0F`)으로 강조 표시됩니다.
