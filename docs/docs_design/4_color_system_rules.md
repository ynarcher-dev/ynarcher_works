# [4] 브랜드 컬러 시스템 및 웹 접근성 규정 정의서

본 문서는 와이앤아처 통합 Works 플랫폼의 브랜드 정체성(CI)을 일관성 있게 전달하고, 정보의 가시성을 높이며, **한국형 웹 콘텐츠 접근성 지침(KWCAG 2.2)**의 명도 대비 규정을 완벽하게 만족하기 위한 **표준 컬러 시스템(Color System)**을 정의합니다.

---

## 1. 브랜드 아이덴티티 컬러 (Brand Colors)
와이앤아처의 대표 브랜드 색상인 CI Red를 중심으로 구성되며, 화면 스크린에서의 피로도를 최소화하기 위해 밝기 단계별로 최적화하여 제한적으로 사용합니다.

### 1.1 브랜드 레드 (Brand Red) Palette
* **대표색 (DEFAULT)**: `#E22213` (정체성 표현 및 포인트 강조 전용)

| 토큰명 | 색상 코드 (HEX) | 사용 가이드라인 및 적용 대상 |
| :--- | :--- | :--- |
| **red.25** | `#FFF5F4` | 아주 연한 레드. 취소/중단 상태의 카드 배경, 에러 알림 배경 |
| **red.500** | `#E22213` | **기본 브랜드 Red**. 활성 탭 인디케이터, 중요 CTA 버튼, 포인트 테두리 |
| **red.800** | `#9F170D` | 어두운 레드. 취소/중단 상태의 텍스트 색상 (KWCAG 4.5:1 명도 대비 충족) |
| **red.900** | `#86130B` | 딥 레드. 브랜드 레드 버튼의 Hover 및 Active 상태 색상 |

* **사이드바 및 메인 본문 표준색**: `#515151` (사이드바 배경 및 메인 텍스트 기본 컬러)

## 3. 상태 신호색 (Semantic Colors) 규칙
시스템 내의 다양한 업무 상태를 시각적으로 직관적으로 전달하는 컬러 가이드입니다. 텍스트 판독성 강화를 위해 **텍스트(Default) - 배경(Subtle) - 보더(Border)**의 3단계 대비 구조를 유지합니다.

### 3.1 세만틱 컬러 적용 매트릭스

#### ① 성공 및 완료 (`Success` ➔ 청색 계열)
* **텍스트**: `#1D4ED8` | **배경**: `#EFF6FF` | **보더**: `#BFDBFE`
* **주요 적용**: 기안 결재 완료, 보육 스타트업 최종 선정, 펀드 결성 완료, 투자 집행 완료 등 완료 및 정상 성공 상태.

#### ② 주의 및 대기 (`Warning` ➔ 황색 계열)
* **텍스트**: `#92400E` | **배경**: `#FFFBEB` | **보더**: `#FDE68A`
* **주요 적용**: 서류 반려, 기한 임박 안내, 승인 검토 대기, 미확인 요청 등 대기 및 주의 상태.

#### ③ 진행 및 정보 (`Info` ➔ 초록 계열)
* **텍스트**: `#166534` | **배경**: `#F0FDF4` | **보더**: `#BBF7D0`
* **주요 적용**: 프로그램 보육 진행 중, 멘토링 매칭 진행 중, 업무 프로젝트 진행 중, 임시 데이터 표기 등 일반적인 진행 및 정보 상태.

#### ④ 취소, 중단 및 위험 (`Danger` ➔ 적색 계열)
* **텍스트**: `#9F170D` | **배경**: `#FFF5F4` | **보더**: `#FAD3CF`
* **주요 적용**: 보육 사업 취소/중단, 투자 계약 반려/철회, 전자결재 문서 회수, 심사 탈락, 차단 상태.

## 4. 웹 접근성(KWCAG 2.2) 컬러 준수 수칙

### 4.1 텍스트 명도 대비 보장
* 화면에 렌더링되는 모든 정보성 텍스트는 배경색 대비 **최소 명도 대비 4.5대 1 이상**을 유지해야 합니다. (단, 18pt/24px 이상의 대형 텍스트는 3대 1 이상).
* 보조 텍스트로 쓰이는 `#4B5563`은 명도 대비 **7.5대 1**로 기준을 충족하며, 포인트 강조용 `#E22213`은 **4.6대 1**로 기준을 만족합니다.
* 비활성 텍스트나 플레이스홀더용 `#9CA3AF`는 명도 대비가 3대 1로 낮으므로, **사용자가 입력해야 하는 안내나 중요 설명 힌트에는 절대로 단독으로 사용하지 않고 보조 설명문을 병행**하여 제공해야 합니다.

### 4.2 색상 단독 구분 지양 원칙
* 정보 취약계층(적록색약 등)이 색상만으로 업무 상태를 혼동하지 않도록 제어합니다.
* **필수 규정**: 성공, 대기, 중단 등을 표현할 때 단순히 뱃지의 컬러(파랑, 노랑, 빨강)만 노출해서는 안 되며, **반드시 상태 텍스트 라벨("완료", "대기", "취소") 또는 형태가 다른 아이콘을 1:1로 함께 노출**하여 조판해야 합니다.

## 5. 컴포넌트별 인터랙션 컬러 규칙 (Component Interaction Rules)
플랫폼의 대표적인 인터랙션 컴포넌트인 버튼과 입력 필드의 상태별 컬러 규격을 정의하여 일관된 조작감을 제공합니다.

### 5.1 버튼 (Button) 컴포넌트
버튼은 역할(Variant)에 따라 아래의 테이을윈드 및 브랜드 테마 토큰 컬러 규격을 엄격히 준수합니다.

* **Primary (주요 액션 — 등록, 저장, 결재상신 등)**:
  * **기본 (Default)**: `bg-brand` (`#E22213`) / 글씨 `text-white`
  * **호버 (Hover)**: `bg-brand-600` (`#C91E11`)
  * **클릭 (Active)**: `bg-brand-700` (`#B31A0F`)
* **Secondary (보조 액션 — 취소, 이전, 닫기 등)**:
  * **기본 (Default)**: `bg-gray-100` (`#F0F0F0`) / 글씨 `text-gray-800` (`#2F2F2F`)
  * **호버 (Hover)**: `bg-gray-200` (`#E5E5E5`)
  * **클릭 (Active)**: `bg-gray-300` (`#D4D4D4`)
* **Outline (테두리 액션 — 파일 다운로드, 목록 필터링 등)**:
  * **기본 (Default)**: `border-gray-300` (`#D4D4D4`) 테두리 + `bg-white` / 글씨 `text-gray-800`
  * **호버 (Hover)**: `bg-gray-50` (`#F7F7F7`)
  * **클릭 (Active)**: `bg-gray-100` (`#F0F0F0`)
* **Ghost (텍스트 액션 — 테이블 행 제어, 간편 링크 등)**:
  * **기본 (Default)**: 배경 없음 / 글씨 `text-gray-700` (`#3F3F3F`)
  * **호버 (Hover)**: `bg-gray-100` (`#F0F0F0`)
  * **클릭 (Active)**: `bg-gray-200` (`#E5E5E5`)
* **Danger (파괴성 액션 — 삭제, 철회, 기각 등)**:
  * **기본 (Default)**: `bg-brand-700` (`#B31A0F`) / 글씨 `text-white`
  * **호버 (Hover)**: `bg-brand-800` (`#9F170D`)
  * **클릭 (Active)**: `bg-brand-900` (`#86130B`)

### 5.2 입력 필드 (Input / Select / TextArea)
사용자가 데이터를 입력하는 폼 요소는 각 포커스 및 검증 실패 피드백이 실시간으로 제공되어야 합니다.

* **기본 상태 (Default)**:
  * `bg-white` 배경 + `border-gray-300` (`#D4D4D4`) 테두리 / 글씨 `text-gray-800` (`#2F2F2F`)
  * 플레이스홀더: `text-gray-400` (`#A3A3A3`)
* **포커스 상태 (Focus)**:
  * 텍스트 입력 활성화 상태. 테두리는 브랜드 포인트 컬러인 `border-brand` (`#E22213`)로 강조하며, 외곽에 투명도 30%의 아웃라인 링(`focus-visible:ring-brand/30`)을 둘러 포커스 위치를 선명히 표시합니다. (기본 outline은 제거)
* **비활성 상태 (Disabled)**:
  * 입력 및 수정 불가 상태. `bg-gray-50` (`#F7F7F7`) 연한 그레이 배경에 `border-gray-200` (`#E5E5E5`) 테두리 / 글씨 `text-gray-400` (`#A3A3A3`)
  * 마우스 포인터는 금지 마크(`cursor-not-allowed`)를 노출하여 사용자의 오인 조작을 방지합니다.
* **오류 상태 (Invalid)**:
  * 필수값 누락 등 유효성 검사 실패 상태. 테두리 및 포커스 링이 경고용 붉은색 (`border-brand-700` / `#B31A0F`)으로 강조 표시됩니다.
