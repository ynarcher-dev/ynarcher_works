# [6] 인터랙션 모션 및 트랜지션 규격서

본 문서는 와이앤아처 통합 Works 플랫폼의 시각적 자연스러움을 확보하고 조작 피드백을 사용자에게 명확히 전달하기 위해, 인터랙션 요소에 적용되는 **전역 모션 및 트랜지션(Duration & Easing) 토큰과 구현 규칙**을 정의합니다.

---

## 1. 모션 설계 원칙 (Motion Principles)

통합 Works 플랫폼 내의 모든 모션은 장식적인 목적을 배제하고, 사용자의 인지 과정을 돕는 기능적 수단으로 사용합니다.

*   **반응성 (Responsiveness)**: 사용자의 조작(클릭, 터치)에 즉각적이고 직관적인 시각적 피드백을 제공하여 시스템이 활성 상태임을 인지시킵니다.
*   **자연스러움 (Natural Flow)**: 화면 요소가 예고 없이 급작스럽게 나타나거나 사라지지 않고, 부드러운 감속/가속 이징을 적용해 사용자의 시선 이동을 자연스럽게 유도합니다.
*   **일관성 (Consistency)**: 유사한 역할을 하는 컴포넌트(모달, 드로어, 드롭다운)는 동일한 속도(Duration)와 곡선(Easing) 토큰을 공유하여 일관된 사용 환경을 유지합니다.

---

## 2. 글로벌 트랜지션 속도 토큰 (Duration Tokens)

인터랙션의 복잡도와 요소의 이동 거리(물리적 크기)에 따라 4가지 표준 속도를 규격화하여 사용합니다.

| 토큰명 | 프레임워크 클래스 (Tailwind) | 밀리초 (ms) | 주요 적용 대상 |
| :--- | :--- | :---: | :--- |
| **`duration.instant`** | `duration-75` | 75ms | 단순 텍스트 컬러 변경, 라디오/체크박스 선택 인디케이터 체크 모션 |
| **`duration.fast`** | `duration-150` | 150ms | **기본 호버(Hover) 효과**, 버튼 활성 상태 전환, 토글 스위치 썸(Thumb) 이동 |
| **`duration.normal`** | `duration-200` | 200ms | 탭 메뉴 언더라인 이동, 드롭다운 팝오버 개폐, 아코디언 펼침/접힘 |
| **`duration.slow`** | `duration-300` | 300ms | 모달 다이얼로그 페이드인, 모바일 사이드바 드로어(Drawer) 슬라이딩 |

---

## 3. 글로벌 가속도 토큰 (Easing Tokens)

이징 곡선은 물리적인 움직임의 현실성을 웹 스크린에 투영하여 부드러운 가감속을 표현합니다.

*   **Standard Easing (`ease-in-out` / `cubic-bezier(0.4, 0, 0.2, 1)`)**:
    *   **설명**: 가장 널리 쓰이는 표준 가감속 곡선. 화면 내에서 상태가 전환되거나 위치가 바뀔 때 사용합니다.
    *   **적용**: 토글 스위치 이동, 탭 이동, 아코디언 및 드롭다운 토글.
*   **Decelerate Easing (`ease-out` / `cubic-bezier(0, 0, 0.2, 1)`)**:
    *   **설명**: 빠른 시작 후 서서히 감속하는 곡선. 화면 외부에서 내부로 컴포넌트가 진입할 때 즉각적인 피드백을 전달하기 위해 사용합니다.
    *   **적용**: 모달 다이얼로그 등장, 모바일 드로어(사이드바) 열림, 툴팁 표시.
*   **Accelerate Easing (`ease-in` / `cubic-bezier(0.4, 0, 1, 1)`)**:
    *   **설명**: 느리게 시작해 빠르게 화면 밖으로 나가는 곡선. 화면 내부의 컴포넌트가 닫히거나 퇴장할 때 사용합니다.
    *   **적용**: 모달 다이얼로그 닫힘, 모바일 드로어 닫힘, 툴팁 사라짐.

## 4. 주요 컴포넌트 모션 및 트랜지션 스펙

### 4.1 모바일 드로어 (사이드바)
*   **진입 (열기)**:
    *   **방향**: 좌측 밖에서 우측 안으로 슬라이딩.
    *   **CSS 규칙**: `transition-transform duration-300 ease-out`
    *   **딤 오버레이 (Dim Background)**: 모바일 사이드바 등장 시 뒷배경 어두워짐 효과도 `duration-300 ease-out` 페이드인을 동일 매핑합니다.
*   **퇴장 (닫기)**:
    *   **방향**: 우측 안에서 좌측 밖으로 슬라이딩.
    *   **CSS 규칙**: `transition-transform duration-300 ease-in`

### 4.2 모달 다이얼로그 (Modal Dialog)
사용자의 집중도가 높은 모달 팝업은 페이드(Fade)와 미세한 스케일(Scale) 변화를 결합해 시각적 볼륨감을 제공합니다.

*   **진입 (열기)**:
    *   **효과**: 배경 딤 오버레이 `opacity: 0 ➔ 1`, 모달 바디 `scale: 95% ➔ 100%` 및 `opacity: 0 ➔ 1`.
    *   **CSS 규칙**: `transition-all duration-300 ease-out`
*   **퇴장 (닫기)**:
    *   **효과**: 배경 딤 오버레이 `opacity: 1 ➔ 0`, 모달 바디 `scale: 100% ➔ 95%` 및 `opacity: 1 ➔ 0`.
    *   **CSS 규칙**: `transition-all duration-200 ease-in` *(퇴장은 더 신속하게 반응하도록 시간 단축)*

### 4.3 드롭다운 & 아코디언
*   **효과**: 세로 높이가 동적으로 변하는 슬라이딩 개폐.
*   **CSS 규칙**: `transition-[max-height,opacity] duration-200 ease-in-out`
*   **화살표 회전**: 펼침 상태 아이콘(Chevron 등)의 회전은 `transition-transform duration-200 ease-in-out`을 동일하게 매핑하여 어긋남 없는 싱크를 유지합니다.
