# [9] 피드백 및 알림 시스템 규격서

본 문서는 와이앤아처 통합 Works 플랫폼 내에서 사용자 조작에 따른 결과를 전달하는 **상태 알림(토스트, 배너), 입력 폼의 실시간 유효성 피드백, 그리고 데이터 조회 지연 시의 대기 상태(로딩, 스켈레톤) 규격**을 정의합니다.

---

## 1. 설계 원칙 (Feedback Principles)

*   **즉각성 (Immediacy)**: 사용자가 등록, 수정, 삭제 등의 중요 액션을 완료하면 1초 이내에 시각적 피드백을 주어 성공 또는 실패 여부를 명확히 학습시킵니다.
*   **방해 최소화 (Non-intrusive)**: 단순 확인용 알림은 화면 중앙을 가리는 모달 대신 화면 주변부에 머무는 토스트(Toast)를 사용하여 조작 흐름이 끊기지 않게 합니다.
*   **다중 채널 전달**: 색각 이상 사용자를 배려하여 상태를 표현할 때는 색상뿐만 아니라 **전용 아이콘과 명확한 피드백 텍스트를 함께 제공**합니다.

---

## 2. 알림 메시지 컴포넌트 명세

### 2.1 전역 토스트 (Toast)
*   **배치 위치**: 화면 우측 상단 (`fixed top-4 right-4`)에 배치하며, 다중 발생 시 위에서 아래로 누적하여 리스트 형태로 노출합니다.
*   **물리 규격**: 가로 고정 `360px` | 모서리 `8px` (`radius.lg`) | 배경색 `#FFFFFF` | 테두리 `border-gray-200` (`#DFE2E7`).
*   **그림자 효과**: `shadow.dialog` (`0 8px 24px rgba(0, 0, 0, 0.12)`) 적용.
*   **생명 주기 (Duration)**:
    *   **Info / Success**: 등장 후 **3초(3,000ms)** 동안 유지된 후 자동으로 우측 방향으로 슬라이드 아웃되며 소멸합니다.
    *   **Warning / Error**: 사용자가 인지하고 수동으로 처리할 수 있도록 자동 소멸 시간을 **5초(5,000ms)**로 연장하거나, 우측 끝에 닫기 아이콘(`X`)을 배치하여 직접 클릭하기 전에는 유지되도록 구현할 수 있습니다.
*   **비주얼 구성**:
    *   좌측 영역에 상태 신호색과 매핑되는 전용 원형 아이콘(`20px`) 배치.
    *   중앙 영역에 볼드체 타이틀(`14px`, `gray.900`) 및 설명문(`12px`, `gray.500`)을 위아래로 결합합니다.

### 2.2 인라인 알림 배너 (Alert Banner)
*   **용도**: 특정 대시보드 세부 구역 상단이나, 설정 콘솔 페이지 최상단에 고정 노출되어 중요 알림(ex: "계정이 연동되지 않았습니다.")을 전파합니다.
*   **물리 규격**: 가로 너비 `100% (Full)` | 상하 패딩 `12px` (`py-3`), 좌우 패딩 `16px` (`px-4`) | 모서리 `6px` (`radius.md`).
*   **컬러 매핑 규칙**:
    *   [4_color_system_rules.md](./4_color_system_rules.md)의 **상태 신호색 3단계 명도 대비(텍스트-배경-보더)**를 그대로 상속합니다. (예: Danger 배너 ➔ 텍스트 `#9F170D`, 배경 `#FFF5F5`, 보더 `#FFE2E2`)

---

## 3. 입력 필드 유효성 검사 에러 피드백

사용자가 폼을 입력하거나 최종 상신 버튼을 누를 때 발생하는 필드 단위 에러 명세입니다.

```
[ 이메일 주소 * ]                           <- 폼 레이블
+---------------------------------------+
| user@domain                           |   <- 입력 필드 (border-brand-700)
+---------------------------------------+
⚠ 올바른 이메일 형식이 아닙니다.            <- 에러 헬퍼 텍스트 (12px, danger.800)
```

*   **입력창 테두리 변화**: 유효하지 않은 값 입력 시, 입력 필드 보더가 즉각적으로 경고용 Red인 `border-danger-700` (`#B31A0F`)로 강조됩니다.
*   **에러 헬퍼 텍스트 (Error Helper Text)**:
    *   **배치**: 입력 필드 하단 `4px` (`mt-1`) 간격을 두고 노출합니다.
    *   **텍스트 규격**: `12px` (`text-xs`) | Regular | `#9F170D` (`danger.800`).
    *   **비주얼**: 텍스트 좌측에 경고용 경고창 아이콘(Alert Triangle, `14px`)을 결합하여 시인성을 극대화합니다.

---

## 4. 대기 상태 로딩 스키마 및 스켈레톤 UI

네트워크 통신 지연 시 사용자가 시스템이 정상 동작 중임을 신뢰할 수 있게 하는 화면 대기 규격입니다.

### 4.1 스피너 (Spinner)
*   **인터랙션**: 무한 회전 루프 모션 (`animate-spin` | `1s` 주기 linear 회전).
*   **크기 표준 스케일**:
    *   **sm**: `16px * 16px` (버튼 내 저장 중 로딩 피드백 표시용)
    *   **md**: `24px * 24px` (카드 영역 단독 데이터 로딩용)
    *   **lg**: `48px * 48px` (전체 페이지 초기 렌더링용 센터 정렬 로딩)
*   **컬러**: 기본 브랜드 네이비 `#1F3A5F` (`bg-brand`) 또는 무채색 그레이 `#6E7683` (`gray.400`)을 적용합니다.

### 4.2 스켈레톤 UI (Skeleton UI)
통합 Works 내에서 1초 이상의 대량 조회가 발생하는 대시보드 차트 영역이나 스타트업 DB 테이블 목록 영역에는 스피너 대신 스켈레톤 UI를 제공해 인지 지연 시간을 낮춥니다.

*   **모션 가이드**: 1.5초 간격으로 밝기가 서서히 변하는 은은한 점멸 펄스 애니메이션을 전역 적용합니다. (`animate-pulse` | `duration-1000 ease-in-out infinite`).
*   **스켈레톤 구성 요소 가이드**:
    *   **바탕색**: `#EDEFF2` (`gray.100`)을 기본 도형의 배경색으로 채웁니다.
    *   **아바타 스켈레톤**: `rounded-full` 원형 기본 도형 배치.
    *   **텍스트/컬럼 스켈레톤**: `h-4` 높이의 가로형 막대 배치, 모서리는 `4px` (`radius.sm`) 적용.
    *   **카드/블록 스켈레톤**: 원래 표시되어야 하는 컴포넌트의 모서리 둥글기(`radius.lg`)를 동일 적용하여 구조적 아웃라인을 투영합니다.
