# [3] 타이포그래피 및 폰트 적용 규칙 정의서

본 문서는 와이앤아처 통합 Works 플랫폼의 시각적 일관성과 정보의 위계(Hierarchy)를 확립하고, 화면 해상도에 구애받지 않고 텍스트의 가독성과 판독성을 보장하기 위한 **표준 폰트 적용 규격**을 정의합니다.

---

## 1. 기본 폰트 패밀리 (Font Family)
다양한 기기 및 웹 브라우저 환경에서 동일한 자형과 일관된 렌더링 품질을 확보하기 위해, 화면 스크린 최적화 폰트를 표준으로 채택합니다.

* **국문/영문 공통 표준 폰트**: **프리텐다드 (Pretendard)**
  * *이유*: 본고딕(Noto Sans CJK KR)의 자간 문제를 해결하고 가독성을 고도로 개선하여, 대형 테이블의 수치 데이터와 정성 피드백 텍스트가 모두 뭉침 없이 깔끔하게 표출되는 최적의 화면용 Sans-serif 폰트입니다.
  * *CSS 선언 예시*:
    ```css
    font-family: 'Pretendard Variable', Pretendard, -apple-system, BlinkMacSystemFont, system-ui, Roboto, sans-serif;
    ```
* **영문/숫자 강조용 서브 폰트**: **인터 (Inter)**
  * *이유*: 대시보드의 대형 대시보드 지표, 펀드 결성액 숫자 등의 수치 데이터 가독성을 강화하기 위해 숫자에 특화된 Inter 폰트를 혼용할 수 있습니다.

---

## 2. 타이포그래피 스케일 (Typography Scale)
화면 구조 상의 중요도와 위계를 한눈에 식별할 수 있도록, 폰트 크기(Font Size)와 행간(Line Height), 자간(Letter Spacing)의 조합 스케일을 다음과 같이 고정하여 사용합니다.

| 분류 | 폰트 크기 (Rem/Px) | 행간 (Line Height) | 자간 (Letter Spacing) | 주요 적용 대상 |
| :--- | :--- | :---: | :---: | :--- |
| **Title Large (대제목)** | `1.875rem` (30px) | `1.3` (39px) | `-0.02em` | 마스터 상세 페이지 최상위 타이틀, 메인 대시보드 타이틀 |
| **Title Medium (중제목)**| `1.5rem` (24px) | `1.35` (32px) | `-0.02em` | 상세 페이지 서브 섹션 타이틀, 중요 데이터 통계 요약 카드 내 숫자 |
| **Title Small (소제목)** | `1.25rem` (20px) | `1.4` (28px) | `-0.015em` | 개별 카드 컴포넌트 헤더, 모달 다이얼로그 타이틀 |
| **Body Large (본문 대)**  | `1.0rem` (16px) | `1.5` (24px) | `-0.01em` | 기안서 본문 내용, 외부 전문가 정성 피드백 목록, 일반 게시글 |
| **Body Regular (본문 기본)**| `0.875rem` (14px) | `1.5` (21px) | `-0.01em` | **기본 본문 텍스트**, 폼 입력 필드 레이블, 테이블 내부 셀 데이터 |
| **Caption (보조/설정)**   | `0.75rem` (12px) | `1.4` (16px) | `0em` | 입력 폼 하단 보조 설명문, 테이블 헤더 레이블, 메타 데이터(날짜 등) |

---

## 3. 폰트 두께 사용 규칙 (Font Weight)
사용자에게 시각적 노이즈를 주지 않으면서도 강조점을 명확히 인지시키기 위해, 폰트 두께는 **3가지 단계**로 제한하여 적용합니다.

1. **Bold (700)**:
   * **용도**: 페이지 제목, 모달 버튼 텍스트, 중요 지표 수치 데이터의 강조.
   * **가이드라인**: 한 화면에 지나치게 굵은 텍스트가 남발되면 가독성이 떨어지므로, 정보 위계의 최상단 요소를 강조할 때만 제한적으로 사용합니다.
2. **Medium (500)**:
   * **용도**: 입력 폼 레이블 명칭, 탭(Tab) 메뉴 글씨, 사이드바 내비게이션 메뉴 목록, 테이블 컬럼 헤더, 카드 컴포넌트 내 소제목.
   * **가이드라인**: 정보의 라벨 역할을 하는 요소에 적용하여 일반 셀 텍스트나 본문 데이터와 시각적으로 명확히 분리되게 합니다.
3. **Regular (400)**:
   * **용도**: 일반 본문 텍스트, 테이블 셀 데이터 콘텐츠, 설명문 텍스트, 플레이스홀더.
   * **가이드라인**: 장문의 텍스트나 대량의 테이블 행 데이터에 적용하여 시각적 피로도를 낮추고 판독성을 극대화합니다.

---

## 4. 텍스트 컬러 위계 및 웹 접근성 (KWCAG) 준수 규정
텍스트의 대비(Contrast)를 활용하여 시각적 정보 위계를 확립하며, **한국형 웹 콘텐츠 접근성 지침(KWCAG 2.2)**의 명도 대비 규정을 엄격히 준수합니다.

텍스트 컬러의 실제 색상 값은 [4_color_system_rules.md](./4_color_system_rules.md)의 Neutral Gray 팔레트를 단일 원천으로 참조합니다.

* **Primary Text (`--text-primary` / `#1F1F1F` (gray.900) - 명도 대비 약 16.5 : 1)**:
  * **설명**: 주 텍스트 색상 (매우 어두운 차콜 그레이). 
  * **적용**: 페이지 타이틀, 폼 레이블, 테이블 본문 텍스트 등 모든 중요 정보 영역.
* **Secondary Text (`--text-secondary` / `#515151` (gray.600) - 명도 대비 약 7.9 : 1)**:
  * **설명**: 보조 및 설명 텍스트 색상 (중간 차콜 그레이).
  * **적용**: 카드 내 메타 설명문, 폼 필드 입력 힌트 안내문, 사이드바 비활성 메뉴. 
  * **접근성**: KWCAG 기준인 **최소 명도 대비 4.5대 1**을 훨씬 초과하여, 저시력자도 보조 설명을 쉽게 판독할 수 있도록 보장합니다.
* **Muted Text (`--text-muted` / `#A3A3A3` (gray.400) - 명도 대비 약 2.5 : 1)**:
  * **설명**: 비활성화 텍스트 색상 (연한 그레이).
  * **적용**: 플레이스홀더 텍스트, 비활성 탭 메뉴, 날짜 표기 등의 장식적 메타 데이터.
  * **접근성 주의**: 명도 대비가 4.5대 1 미만이므로, **사용자가 반드시 읽어야 하는 안내나 필수 입력 필드의 힌트 정보에는 단독으로 사용하지 않습니다.**
* **Brand Highlight (`--text-brand` / `#E22213` - 명도 대비 4.6 : 1)**:
  * **설명**: 브랜드 포인트 강조 색상.
  * **적용**: 활성화된 내비게이션 메뉴 텍스트, 중요 링크. 흰색 배경 대비 최소 4.5대 1 이상을 보장하여 시인성을 확보합니다.

---

## 5. 한국 디지털 서비스 가이드라인 기반 조판 및 가독성 원칙
정부 디지털 서비스 가이드라인 및 한글 조판 원칙에 근거하여 국문 텍스트 판독성을 극대화합니다.

1. **텍스트 크기 가변성 보장 (200% 확대 대응)**:
   * 사용자가 브라우저 글자 크기를 조절하거나 화면을 200% 확대하더라도 텍스트가 겹치거나 잘리지 않고 레이아웃이 유동적으로 반응할 수 있도록, 폰트 크기 정의 시 절대 단위(`px`) 대신 **상대 단위(`rem`) 사용을 기본 표준**으로 합니다.
2. **시스템 폰트 크기 최솟값 제한**:
   * 정보 취약계층 및 모바일 환경에서의 최소 판독성을 보장하기 위해, 화면에 노출되는 가장 작은 텍스트(예: 캡션, 뱃지 등)라도 **최소 `0.75rem` (12px) 미만으로 설정되는 것을 금지**합니다.
3. **한글 자간 최적화**:
   * 국문 프리텐다드 서체의 경우, 글자 사이의 불필요한 여백을 메워 단어 인지 속도를 높일 수 있도록 CSS 자간 설정을 `-0.01em`~`-0.02em` 수준으로 미세 조정하여 렌더링합니다.
4. **가독성 행간 및 줄바꿈 표준 준수**:
   * **행간 (Line Height)**: 본문 텍스트 가독성 확보를 위해 최소 1.5배~1.6배의 행간을 보장합니다.
   * **줄바꿈 규칙**: 텍스트 영역 내에서 명사나 단어가 강제로 잘리지 않도록 `word-break: keep-all;` 및 단어 단위 줄바꿈 시 레이아웃 침범을 막는 `overflow-wrap: break-word;` 속성을 상시 결합하여 조판합니다.
