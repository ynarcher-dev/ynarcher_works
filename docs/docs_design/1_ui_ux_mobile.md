# [1] UI/UX 및 모바일 최적화 정책서

본 문서는 출장과 현장 업무가 잦은 심사역과 외부 게스트의 접근성을 고려하여 수립한 **모바일 반응형 레이아웃 및 간편 인증(OTP/일회용 링크) UX 정책**을 다룹니다.

---

## 1. 반응형 레이아웃 정책 (Responsive Layout Policy)

통합 Works 플랫폼은 단일 웹 애플리케이션으로 개발되되, 모바일(Mobile), 태블릿(Tablet), 데스크톱(Desktop) 디바이스에서 최적의 가독성과 제어 편리성을 보장하기 위해 자동 리플로우 레이아웃 규칙을 채택합니다.

### 1.1 해상도 기준점 (Breakpoints)
* **모바일 (Mobile)**: 360px ~ 767px (스마트폰 포트레이트 모드 최적화)
* **태블릿 (Tablet)**: 768px ~ 1023px (패드 기기 뷰포트 대응)
* **데스크톱 (Desktop)**: 1024px 이상 (사무실 모니터 표준 해상도 대응)

### 1.2 모바일 레이아웃 자동 변환 규칙
* **사이드바 내비게이션 (Sidebar to Hamburger)**:
  * 데스크톱 화면에서 항상 좌측에 고정되어 노출되던 메인 내비게이션 사이드바는 모바일 해상도 진입 시 자동으로 화면 좌측 영역 밖으로 접힙니다.
  * 헤더 영역 좌측 상단에 **햄버거 버튼(☰)**이 활성화되며, 터치 시 부드러운 슬라이딩 애니메이션과 함께 내비게이션 드로어가 열리도록 구성합니다.
* **그리드 레이아웃의 단일 열 리플로우**:
  * 데스크톱에서 가로로 배치되던 2열 또는 3열 레이아웃 컴포넌트는 모바일 뷰포트에서 자동으로 **1열 세로 배치**로 떨어져 위에서 아래로 스크롤하여 읽을 수 있도록 변환됩니다.
* **복잡한 데이터 테이블의 모바일 간편화**:
  * 가로 폭이 좁은 모바일 화면에서 열(Column)이 많은 데이터 테이블을 그대로 노출할 경우 가독성이 심각하게 훼손됩니다.
  * 따라서 모바일 뷰포트에서는 주요 열(기업명, 상태, 금액 등) 2~3개만 테이블로 노출하고, 나머지 세부 항목은 각 행을 터치했을 때 아래로 슬라이드 다운(Slide Down)되어 펼쳐지는 **아코디언(Accordion) 방식** 또는 **요약 정보 목록 뷰**로 변환 렌더링합니다.

## 2. 모바일 특화 제어 요소 (Mobile UX)

* **스와이프 제어 (Swipe Gesture)**:
  * M&A 칸반 보드 등 가로 스크롤이 필수적인 공간에서는 모바일 터치 스와이프를 지원하여, 화면을 손가락으로 가볍게 쓸어 넘기며 단계를 쉽게 전환할 수 있게 만듭니다.
* **모바일 최적화 승인 버튼**:
  * MANAGEMENT 워크스페이스의 전자결재 등 즉각적인 조치가 필요한 화면은 모바일 화면 하단에 플로팅 액션 버튼(Floating Action Button) 형태로 결재 승인/반려 영역을 고정 노출하여, 한 손으로 터치하여 결재를 신속하게 처리할 수 있도록 설계합니다.
