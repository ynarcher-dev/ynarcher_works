# [2] 공통 레이아웃 및 내비게이션 기획 명세서

본 문서는 와이앤아처 통합 Works 플랫폼의 최상위 화면 프레임인 공통 쉘(AppShell), 상단바(Topbar), 좌측 사이드바(Sidebar)의 UI/UX 구성 요소 및 사용자의 이동 흐름을 통제하는 내비게이션 규칙을 정의합니다.

---

## 1. 레이아웃 그리드 및 프레임 개요
플랫폼의 전체 레이아웃은 사용자가 데스크톱, 태블릿, 모바일 등 다양한 기기에서 일관된 경험을 유지할 수 있도록 3분할(사이드바 + 상단바 + 본문 콘텐츠) 구조를 기본으로 설계하며, 화면 공간을 좁게 고정하지 않고 넓게 활용할 수 있도록 유연한 풀-와이드(Full-Width) 컨테이너 및 1열/2열 가변 그리드 시스템을 적용합니다.

```
+---------------------------------------------------------+
| [CI LOGO] | 상단바 (높이: 64px)                          |
|           | [서비스 전환 ▾]                     [프로필 ▾] |
+-----------+---------------------------------------------+
| 좌측      | 본문 콘텐츠 영역                             |
| 사이드바  | (1열형: Full-Width 테이블/차트 화면)          |
|           | (2열형: 좌측 기본 카드 + 우측 연관 패널)     |
| (240px)   |                                             |
|           |                                             |
+-----------+---------------------------------------------+
```

* **좌측 사이드바 (Sidebar)**: 데스크톱 화면에서 가로 `240px` 고정 폭으로 화면 좌측에 고정되며, 현재 워크스페이스의 세부 메뉴 목록을 표시합니다.
* **상단바 (Topbar)**: 세로 `64px` 높이로 상단에 고정되며, 워크스페이스 간 이동(서비스 전환) 드롭다운, 알림, 통합 기능 트리거, 사용자 정보 메뉴를 배치합니다.
* **본문 콘텐츠 영역 (Content)**: 상단바와 사이드바를 제외한 나머지 화면 영역으로, 좌우 여백을 균형 있게 주는 동시에 데이터 밀도가 높은 화면에서는 넓게 쓰며 컨텍스트에 따라 아래의 **1열/2열 분할 규칙**을 적용합니다.

### 1.1 콘텐츠 그리드 분할 규칙 (1열 & 2열 구성)

#### ① 1열형 레이아웃 (Full-Width Layout)
* **적용 대상**: 대시보드 리포트(차트 나열형), 통합 검색 결과 목록, 대량의 행이 존재하는 리스트/테이블 화면(예: 스타트업 DB 리스트, 감사 로그 등).
* **특징**: 화면 폭을 최대 가용 영역까지 넓게 사용하여, 가로 스크롤을 최소화하고 대형 테이블의 다중 컬럼 데이터를 한눈에 시원하게 판독할 수 있게 합니다.

#### ② 2열형 레이아웃 (Two-Column Layout)
* **적용 대상**: 스타트업/전문가 상세 프로필 조회 화면, 보육 프로그램 매칭 캘린더, M&A 딜 검토 상세 및 전자결재 문서 상세 화면.
* **특징**: 화면을 크게 **주 콘텐츠(Primary)**와 **보조 정보(Secondary)**로 분할하여 정보의 다각적 탐색을 유기적으로 돕습니다.
  * **주 콘텐츠 열 (좌측, 65% ~ 70% 비율)**: 주요 핵심 인적 정보 카드, 기본 업무 폼, 메인 기안 문서 등 사용자 집중이 필요한 핵심 내용을 표출합니다.
  * **보조 정보 열 (우측, 30% ~ 35% 비율 또는 고정 360px~400px)**: 연관 파일 첨부 패널, 히스토리 로그 타임라인, 정성 피드백 의견 카드(아코디언 형태), 혹은 결재선 지정 박스 등을 함께 배치합니다.
* **반응형 흐름(Responsive Behavior)**:
  * 화면 가로 폭이 충분한 데스크톱에서는 2열로 나란히 배치하여 화면을 넓고 입체적으로 씁니다.
  * 가로 폭이 부족한 태블릿 및 모바일 환경(1024px 미만)에서는 **우측 보조 열이 좌측 주 콘텐츠 열 아래로 흐르는(Stack) 1열 형태로 자동 적응**하여 모바일 스크롤만으로 모든 정보를 유기적으로 조회할 수 있도록 대응합니다.

## 2. 상단바 (Topbar) 상세 요건

### 2.1 서비스 전환 드롭다운 (Workspace Switcher)
상단바 좌측 영역에 배치되어 사용자가 다른 워크스페이스로 매끄럽게 이동할 수 있도록 지원하는 핵심 컴포넌트입니다.

* **동작 및 비주얼**:
  * 클릭 전에는 현재 진입한 워크스페이스명(예: `OFFICE`, `STARTUP`, `AC`, `FUND` 등)과 드롭다운 화살표(`▾`) 아이콘이 표시됩니다.
  * 클릭 시 하단으로 드롭다운 형태의 레이어가 부드럽게 펼쳐집니다.
* **권한 기반 노출 제어**:
  * 사용자의 권한 정보(`PermissionMap`)를 읽어와, 해당 워크스페이스(도메인)에 **최소 '읽기(Read)' 권한**이 부여된 공간만 목록에 노출합니다.
  * 권한이 아예 없는 워크스페이스(예: 일반 직원이 볼 수 없는 `ADMIN`이나 타 부서에 잠긴 `M&A`)는 드롭다운 메뉴 목록에서 완전히 숨김(Hidden) 처리하여 무분별한 링크 노출을 방지합니다.
  * `GUEST`는 권한 도메인에는 포함되지만 WORKS 내부 워크스페이스가 아니므로, 서비스 전환 드롭다운에는 노출하지 않고 별도 서브도메인 앱에서 운영합니다.
* **현재 위치 표시**:
  * 드롭다운 목록 중에서 현재 사용자가 머물고 있는 워크스페이스 항목에는 브랜드 포인트 컬러(CI Red 등) 및 우측에 '현재' 텍스트/체크 배지를 노출하여 직관성을 확보합니다.

### 2.2 사용자 프로필 및 계정 관리 메뉴
상단바 우측 끝에 배치되어 현재 로그인한 사용자의 정보를 보여주고 로그아웃 등의 전역 명령을 수행합니다.
* **사용자 아바타**: 사용자의 이름 첫 글자 또는 지정된 이미지를 원형 아바타로 표현합니다.
* **계정 세부 드롭다운**: 아바타 클릭 시 노출되며 사용자명, 이메일 주소, '내 계정 관리' 메뉴 링크, 그리고 명확한 '로그아웃' 버튼을 포함합니다.

## 3. 좌측 사이드바 (Sidebar) 상세 요건

### 3.1 워크스페이스별 동적 메뉴 스위칭 규칙 (Contextual Sidebar Menu)
상단바 드롭다운에서 선택된 워크스페이스(컨텍스트)에 종속된 하위 메뉴군만 좌측 사이드바에 동적으로 노출합니다. 사용자는 하나의 플랫폼 안에서 불필요한 노이즈 없이 현재 업무에만 집중할 수 있습니다.

#### ① OFFICE (전사 업무 허브) 선택 시
* **기본 업무 영역**: 대시보드, AI 에이전트, 임직원 정보, 전사 캘린더, 회의실 예약, 전자결재, 거래처 정보.
* **게시판 영역**: 고정 게시판(공지사항, 공용자료실 등)과 일반 게시판(인사이트 및 신규 생성 게시판)을 런타임 게시판 레지스트리에서 동적으로 주입합니다.

#### ② STARTUP (스타트업 풀) 선택 시
* **현황 영역**: 대시보드.
* **구분별 목록**: 투자기업, 보육기업, 발굴기업, 기타기업. 네 목록은 `management_status` 코드로 사전 필터된 상호 배타 뷰입니다.
* **운영 도구**: 회의록, 아처스캔, 대용량 업로드.

#### ③ NETWORKS (네트워크 원장) 선택 시
* **대시보드**: 네트워크 규모 KPI, 구분별 분포, 최근 업로드 현황.
* **마스터 네트워크 관리**: BAN, EXP, 전문가, 투자사, 기업, 기관, 대학, 외주/거래, 기타 네트워크.
* **데이터 관리**: 글로벌 네트워크, 미분류 데이터베이스, 대용량 업로드.

#### ④ AC (사업 관리) 선택 시
* **AC 메인**: 대시보드.
* **사업 목록**: 내 사업, 전체 사업. 프로그램 상세 내부에서 운영 모듈 카드 클릭으로 모집, 평가, OT, 멘토링, 매칭, 데모데이, 타임라인, 성과, 커스텀 활동 화면으로 진입합니다.

#### ⑤ FUND (투자 관리) 선택 시
* **FUND 메인**: 투자 대시보드
* **출자 관리**: 조합/펀드 결성액 내역, LP(출자자) 비율 관리
* **투자 집행**: 포트폴리오 투자 내역, 캐피탈 콜(Capital Call) 납입 현황

#### ⑥ M&A/PE 선택 시
* **딜 관리**: 딜 소싱 칸반, 매칭 매트릭스.

#### ⑦ PROJECT 선택 시
* **프로젝트 메인**: 프로젝트 대시보드. 상세 화면에서는 태스크 칸반과 간트 로드맵을 함께 탐색합니다.

#### ⑧ MANAGEMENT 선택 시
* **경영지원**: 대시보드, 조직 관리, 지사 관리, 직책 관리, 직급 관리, 호봉 관리, 인사 관리, 자산 관리, 재무 관리, KPI 관리.

#### ⑨ ADMIN (시스템 관리자) 선택 시
* **보안/권한 제어**: 역할(Role) 및 권한 매트릭스 콘솔, 권한 템플릿 관리
* **시스템 기준값 관리**: 게시판 관리, 산업/분야/구분/권역/국가/투자단계/기업구분/기업현황/소재지 태그 관리, 민감정보 관리
* **데이터 정리**: 데이터 중복 병합 검증(NETWORKS 마스터 대상)
* **로그 추적**: 권한 변경 및 데이터 조작 감사 로그, 파일 다운로드 사유 로그 감사

### 3.2 내비게이션 활성화 및 하이라이트 규칙
* 현재 접속 중인 브라우저의 URL 경로(`pathname`)와 사이드바 메뉴 항목의 `href` 경로를 비교하여 일치하거나 하위 경로인 경우 활성화(`active`)로 판정합니다.
* 활성화된 메뉴는 배경색을 `bg-brand-25`(`#FFF5F4`)로 은은하게 채우고, 폰트 색상을 브랜드 메인 컬러로 변경하여 사용자의 현재 위치 인지력을 향상시킵니다. (사이드바 배경은 밝은 배경(`gray.0`/`gray.50`)을 표준으로 합니다.)

## 4. 반응형 모바일 뷰포트 인터랙션

* **데스크톱 해상도 (`1024px` 이상)**:
  * 좌측 사이드바가 고정 형태로 항상 유지됩니다. 본문 영역은 사이드바 너비를 제외한 영역을 가득 채웁니다.
* **모바일 및 태블릿 해상도 (`1024px` 미만)**:
  * 좌측 사이드바는 화면에서 완전히 숨겨집니다.
  * 상단바 좌측에 **햄버거 메뉴 아이콘(Menu Bar Icon)**이 노출됩니다.
  * 사용자가 햄버거 메뉴 아이콘을 터치하면, 화면 좌측에서 사이드바가 슬라이딩 아웃(Drawer) 형식으로 부드럽게 미끄러지며 펼쳐집니다. 사이드바 바깥 영역은 `rgba(0,0,0,0.4)` 배율의 어두운 오버레이 딤(Dim) 처리를 적용하여 본문 영역을 터치하면 다시 닫히도록 유도합니다.
